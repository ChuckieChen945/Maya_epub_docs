transfer_properties_by_2d_indices

将几何特性从多个源几何体传递到单个目标几何体。当从许多现有几何体对象构建新的几何体对象并仍保留现有几何体的自定义几何特性时，这很有用。将复制几何特性值。不执行插值或混合。indices2d 端口用于指定如何将源中的几何特性值映射到目标。

输入

target_geometry

几何特性将被复制到的目标几何体。

source_geometries

将从中复制几何特性的源几何体。

component

要传递几何特性的几何组件。

property_expression

将传递与此表达式匹配的所有特性。

block_schema

此模式中的所有特性将不会被传递。当所有需要传递的特性都是自定义特性时，这将非常有用。例如，在点几何体之间传递特性时，可以通过在此处连接点原型对象（通过 create_object_from_schema 创建的对象）来跳过 point_position 特性。

always_transfer_expression

与此表达式匹配的任何特性将始终被传递，即使它与 block_schema 对象中的特性匹配也是如此。

concatenate

当 concatenate 设置为 true 时，将忽略 indices_2d，并且来自每个源的所有值将连接在一起并传递到目标几何体。例如，如果有 3 个源几何体，分别包含 2、3 和 4 个点，则传递的几何体的每个传递的特性将包含 9 个值。

indices_2d

此数组指定源组件和目标组件之间的映射。每个元素都是一对索引。第一个索引指示要从哪个源几何体传递，第二个索引指示要从指定源复制的元素。例如，如果有 3 个源几何体，每个源几何体包含 4 个点，并且所需传递的值对应于第一个几何体中的点 3、第二个几何体中的点 2 和第三个几何体中的点 0，则此数组应包含 [[0,3], [1,2], [2,0]]

输出

target_with_properties

包含已传递特性的目标几何体。

property_names

已传递特性的名称。
