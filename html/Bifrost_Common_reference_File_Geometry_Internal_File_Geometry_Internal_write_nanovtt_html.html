write_nanovtt

此节点将 VTT 对象写入 NanoVTT 文件。

输入

objects

要写入的 Bifrost 对象。

enable

如果为 true，则启用写入文件。否则，节点将以穿过方式工作。

filename

文件名。

directory

目录。

frame

帧编号。

overwrite

如果为 true，将覆盖现有文件。

create_directories

如果为 true，则创建输出目录层级（如果不存在）。

properties

要写入的特性的列表（以空格分隔）。

quantization_bits

用于量化的位数。如果为负，则禁用量化。如果为零，则使用可变位长量化。

quantization_tolerance

用于量化值的容差。

enable_dither

如果为量化值启用了抖动，则为 True。

extended_tile_fraction

允许进行采样的冗余数据占用的额外空间的比例（介于 0 和 1 之间）。比例越大，需要的空间就越多，但线性采样速度会更快。

store_tile_data_as_3d_textures

将每个特性的平铺数据打包为 3D 纹理。注意：当前仅当 extended_tile_fraction 为 1 且 coalesce_tile_data 和量化处于禁用状态时才启用。

coalesce_tile_data

将大小为 4x4x4 的数据平铺合并为更大的数据平铺，如 8x8x8、6x16x16 等，以提高内存效率。注意：如果已启用量化，则当前处于禁用状态。

输出

out_objects

使输入对象穿过。

success

如果已成功写入文件，则为 true。

file_path

已写入文件的完整路径。
