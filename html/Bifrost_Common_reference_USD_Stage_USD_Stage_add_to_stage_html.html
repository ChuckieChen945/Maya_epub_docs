add_to_stage

将一个或多个 Bifrost-USD 基本体定义添加到新的或现有 USD 后台文件。这将在后台文件上创建 USD 基本体。

输入

stage

作为添加目标的 USD 后台文件。如果未连接任何对象，将创建空后台文件。可以在端口上单击鼠标右键，然后选择“连接节点”(Connect Node)以快速添加和连接可打开或创建新后台文件的节点。

prim_definitions

要添加的 Bifrost-USD 基本体定义列表。可以在端口上单击鼠标右键，然后选择“创建节点”(Create Node)以快速添加和连接任意常用输入。

Settings

这些设置将应用于所有添加的基本体。

enable

禁用此选项可跳过添加基本体的步骤，并改为输出未修改的后台文件。

parent_path

将连接的基本体作为子对象添加到的基本体的路径。例如，如果这是 /set/geo/，且连接的基本体定义具有路径 /street/car，则基本体在后台文件中将变为 /set/geo/street/car。绝对路径需要前导斜杠。

  • 如果省略 parent_path，则基本体将添加到根目录。
  • 如果后台文件中尚不存在完整路径中的一个或多个基本体，则使用 def 说明符将“占位符”基本体自动添加到层次中。可以使用其他 add_to_stage 节点在图表中的下游更改这些说明符。

layer_index

要向其添加基本体定义的层的索引。子层按顺序索引，0 位于堆栈底部，每个后继层位于上一层上方。0 层包含最弱的观点，这与使用 USD Python 或 C++ API 时子层的排序方式相反。默认值“-1”表示根层。默认值“-1”表示根层。如果指定的索引不存在，则使用层“-1”。

请注意，无法按名称指定 Bifrost-USD 后台文件中的层，因为该后台文件位于内存中，而不是磁盘中。标识符由随机哈希（后跟一个可选的可读标记）组成，并且每次执行图表时哈希都可能发生变化。

use_material_name_or_tag

将具有材质标记的基本体指定给具有匹配标记的材质。如果没有材质具有匹配标记，则此选项会尝试将基本体标记与材质名称相匹配。如果仍然无法匹配，则不指定任何材质。

输出

out_stage

具有添加的基本体定义的后台文件。
