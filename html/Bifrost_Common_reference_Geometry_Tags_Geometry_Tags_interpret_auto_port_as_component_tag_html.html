interpret_auto_port_as_component_tag

将各种输入转化为 array<bool> 和 array<long> 类型。使用此工具节点可以构建支持多种输入类型的可重用复合，以选择几何体上要操作的点或其他组件。例如，可以使用它指定要在 delete_points 复合中删除的点。

要使用此节点，请将 interpreted_tag_port 连接到复合内类型为 auto 的输入，然后在每个组件的计算中使用输出 tag_data 和 tag_indices。

支持以下多种可能的输入类型：

  • array<bool> 解释为每个元素（例如，每个点）的值。数组将会传递，但可调整大小以匹配 geometry 输入上的元素数。
  • array<long> 或 array<uint> 解释为 target 分量类型的索引。
  • long 解释为 target 分量类型的单个索引。
  • string 解释为现有标记的名称或标记表达式。有关标记表达式语法的更多详细信息，请参见有关 resolve_tag_expression 的文档。

输入

geometry

要在其中解释输入的几何上下文。例如，使用默认设置和基于点的几何体时，将输出逐点数据。输出与几何体中的元素（如点）数匹配。

interpreted_tag_port

将此端口连接到复合的 auto 输入，以支持不同类型的输入。从这个端口创建新端口时，还会向该端口添加解释的自动类型调整。

default_value

当字符串输入指定特性但该特性不可用时要使用的默认值。

geometry_component

如果数据上下文不是默认的 point_component，则可以在此处选择其他组件。这是可确定输出 data 数组大小的组件。

invert

设置为 true 将反转输出的标记数据。

输出

tag_data

将输入 interpreted_tag_port 转化为 array<bool> 后所得的结果。此数组与指定 target 类型的 geometry 中的元素数匹配。

tag_indices

将输入 interpreted_tag_port 转化为 array<long> 后所得的结果。

has_data

当字符串连接到 interpreted_tag_port 时，如果在 geometry 上找到标记，则此输出为 true，否则为 false。当 array<bool> 连接到 interpreted_tag_port 时，如果数组为非空，则此输出为 true。当 array<long> 连接到 interpreted_tag_port 时，此输出始终为 true。
