create_skeleton_from_paths

从解释为路径的无序 array<string> 构造骨架。这可用于构造骨架的拓扑数据，无需处理 joint_parent_index 索引和关节数据排序。

路径使用正斜线 / 作为起始字符和实体名称之间的分隔符。

此节点对路径进行排序，使父关节位于其子关节之前，此外还过滤潜在的重复项，使每个关节路径都唯一。如果两个或更多路径相冲突，则将保留要注册的最后一个重复路径以用于构造骨架。

假定以下输入 paths：

paths[0] = /joint0/joint1/joint2
paths[1] = /joint0/joint1
paths[2] = /joint0
paths[3] = /joint0/joint1

构造的骨架的关节数据数组以及返回的 joint_path 将按如下所示排序：

joint_path[0] = /joint0
joint_path[1] = /joint0/joint1
joint_path[2] = /joint0/joint1/joint2

骨架的 joint_parent_index 将按如下所示构造： 

[4294967295, 0, 1]

sorted_indices 输出数组将返回以下索引： 

[2, 3, 0]

请注意输入 paths[1] = /joint0/joint1 是如何被忽略的。原因在于 paths[3] = /joint0/joint1 覆盖已注册的路径，因为它是第一个路径的重复项。因此，如果两个或更多路径相冲突，则将保留要注册的最后一个重复路径以用于构造骨架。

输入

paths

包含用于构造骨架拓扑的路径的 array<string>。

输出

skeleton

构造的骨架对象。

sorted_indices

排序和过滤的索引，这是按字母顺序对 paths 进行排序并过滤其中任何重复项的结果。
