将 Bifrost-USD 后台文件保存到文件中

使用 save_usd_stage 节点可将 Bifrost 后台文件的所有修改层保存到 USD 文件中，包括根层和任何子层。

  1. 将 save_usd_stage 节点添加到图表。通常，应当在修改后台文件的最后一个节点和位于图表顶层的 output 节点之间连接，除非您有意要保存来自更上游的后台文件版本。
    • 连接后台文件，以保存到 stage 输入。
    • 将 out_stage 端口连接到图表顶层的 output 节点，或连接到其下游的任何其他节点。
  2. 如果需要，在 file 中输入文件位置和名称，以覆盖后台文件的文件路径。如果为空，则使用 create_usd_stage 节点的 layer 参数或 open_usd_stage 节点的 save_file 参数中的路径和文件名保存后台文件。
  3. 建议启用 relative_path。这会将对后台文件子层的引用存储为相对于根层文件位置的路径，从而更容易将所有层文件一起移动或复制。
  4. 此外，建议取消选中 enable，以避免在修改图表时持续重新保存，除非您要执行简单测试并希望在外部文本编辑器中实时查看更新。
    • 准备好保存层之后，请打开 Enable，然后再次关闭。
    • 或者，可以连接图表中的 enable 端口，以便在满足特定条件时自动保存。

作为 save_usd_stage 的替代方案，您还可以使用 export_stage_to_file。此节点可以展平组合的后台文件。
