compound_instance

创建通过 Bifrost 复合文件输出的几何体的实例形状。此节点仅在使用 Arnold 进行渲染时适用，它可在渲染时执行 Bifrost 复合。

输入

compound_file

包含复合定义的 JSON 文件的路径和名称。

compound_URI

要用作实例的复合的名称空间和名称，例如 User::My_Geometry。

property_port_schema

指定点对象（连接到下游的 create_instances 或类似节点）的特性名称的字符串数组，这些特性要映射到上面引用的复合上的同名端口。要将特性映射到名称不同的端口，请使用语法 property -> port。

实例

preview_geometry

要用于预览的可选几何对象。如果连接了对象，它将仅显示在视口中，而不用于渲染。

name

可选标识符。

transform

要应用于实例化几何体的可选变换矩阵。

渲染

operators

要应用的任何 Arnold 运算符（可选）。可以将运算符添加到图表中，方法是从大纲视图拖动运算符（当“显示”>“仅 DAG 对象”(Display > DAG Objects Only)处于禁用状态时），然后将其连接到相应端口。

operators

Arnold 运算符可以插入此处，以应用于生成的几何体。

输出

compound_instance

可以连接到 create_instances 或类似节点的实例形状。
