write_OpenVDB_points

此节点将 Bifrost 点对象写入 OpenVDB 文件。

输入

points

要写入的 Bifrost 点对象。

enable

如果为 true，则启用写入文件。否则，节点将以穿过方式工作。

filename

文件名。使用 # 包含输入 frame 编号。# 的出现次数确定填充量。例如，file####.vdb 解析为 file0001.vdb、file0002.vdb 等。

directory

目录。

frame

帧编号。您可以将 time 节点的 frame 输出连接到 to_long 节点，然后在此处连接结果。

overwrite

如果为 true，将覆盖现有文件。

create_directories

如果为 true，则创建输出目录层级（如果不存在）。

properties

要写入的特性的列表（以空格分隔）。使用 *（星号）指定所有特性。另请参见下面的限制。

输出

out_points

使输入点云穿过。

success

如果已成功写入文件，则为 true。

file_path

已写入文件的完整路径。

限制

对于体积，支持 OpenVDB 标准类型的以下子集：float、double、int、long、int3、float3 和 double3。不支持 boolean 和 string 类型。

对于点对象，支持所有 OpenVDB 标准类型的逐点特性。这包括 boolean、float、double、int、long、string、int3、float3 和 double3。

除点对象外，还支持全局特性 - 这些特性对整个对象使用单个数据值，而不是每点使用一个值。例如，这对于存储对象变换可能很有用。全局特性将作为元数据存储在 OpenVDB 栅格中。支持元数据的所有标准 OpenVDB 类型：bool、float、double、int32、int64、string、int2、float2、double2、int3、float3、double3、float4x4 和 doubl4x4。

请注意，体积不支持全局特性。
