将当前选择导出到 MaterialX

导出 MaterialX 外观供 Arnold 使用。

您可以从 Arnold 将着色器树和形状特性导出到 MaterialX 外观。导出内容包括：

  • 曲面着色、置换和体积。
  • “visibility”、sideness、matte 和 displacement 设置等特性（形状参数）会自动导出，并且您可以指定其他形状参数。
  • 有一个类似于位掩码等效项的特殊可见性指定，用户可以通过其指定要禁用摄影机光线。

Arnold 可以在不同的 DCC 中重用或共享导出的外观。对于每个 mtlx 文档，您可以定义导出场景的哪些部分以及外观的名称。Arnold 插件将转换场景，然后调用公共 API 函数，该函数会遍历整个场景、生成 mtlx 文档，然后将其写入文件。

支持：

  • 导出材质指定以及节点图表（包含 Arnold 的专有闭包类型、通道信息和上下文定义（着色器和置换））。
  • 材质/特性/可见性指定。
  • 替换/附加 mtlx 文档模式。

不支持：

  • 外观变体，因为这是场景的烘焙版本。
  • Materialx 标准库支持。
  • 灯光的着色器指定。
  • 其他渲染器和 MaterialX 查看器。

无法在导出到 MaterialX 的同时进行渲染。

环境变量

默认情况下，导出的 .mtlx 文档使用 Arnold 附带的节点定义。用户可以使用环境变量“ARNOLD_MATERIALX_NODE_DEFINITIONS”和“ARNOLD_MATERIALX_SEARCH_PATH”指定自己的节点定义和搜索路径。

文件名(Filename)

选择 mtlx 文件的保存名称和路径。

对于现有的 mtlx 文件，导出将更新现有外观，或者向文档附加新外观。

外观名称(Look Name)

导出的 MaterialX 文档要使用的外观名称。

特性

要导出的其他形状参数。例如，用于渲染为体积的多边形网格的 step_size 和 _volume_padding_。

以下形状参数会自动导出（如果它们设置为非默认值）：

  • visibility
  • sidedness
  • disp_padding
  • disp_height
  • disp_zero_value
  • disp_autobump
  • autobump_visibility

相对指定(Relative Assignments)

如果设置此项，则着色器/特性/可见性指定将相对于形状所在的名称空间，否则，指定将包含形状的完整路径。例如，如果名为“sphere1”的球体位于程序“my_proc”中，则完整名称应为“my_proc/sphere1”，相对名称应为“sphere1'”。这样，可以使用其他名称将外观指定给其他程序。
