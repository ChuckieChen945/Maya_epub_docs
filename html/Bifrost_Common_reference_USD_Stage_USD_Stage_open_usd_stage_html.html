open_usd_stage

将 USD 文件作为后台文件打开。usd（通用）、usda（ASCII 文本）和 usdc（创建二进制）格式支持读取和写入，usdz 文件仅支持读取。

输入

file

要作为后台文件打开的文件。

read_only

防止修改后的后台文件被节点下游保存或导出时覆盖输入文件。

  • 选中此选项时，必须创建一个新后台文件，然后从输入后台文件添加子层或参照基本体到新后台文件。
  • 取消选中该选项时，则应在 save_file 中指定其他路径。

save_file

禁用 read_only 时，这是用于按节点下游保存后台文件的文件路径和名称。启用 read_only 时，该值将被忽略。确保此路径与输入文件不同，以避免循环。

mask

连接基本体路径（字符串）数组，将后台文件填充限制为指定基本体及其子孙和祖先对象。请注意，由于后台文件遮罩未存储在 USD 层中，因此需要共享 Bifrost 后台文件才能在主机应用程序中查看效果。如果主机正在子分层或参照 Bifrost 后台文件的根层，则不会考虑遮罩。

load

控制 USD 有效负载圆弧的行为。

  • LoadAll：加载所有可载入的基本体。
  • LoadNone：记录有效负载圆弧，但不会遍历。

输出

stage

打开的后台文件。
