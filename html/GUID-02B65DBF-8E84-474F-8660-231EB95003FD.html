使用空白

可以通过适当地使用空白和注释改进 MEL 脚本的格式。这两种方法都可使脚本更易于阅读和理解。

在关键字与变量之间至少必须插入一个空格。除此以外，空白仅用于将脚本组织为易读格式。

空白包含空格、制表符和空行。将空白添加到脚本后，不会影响该脚本的执行。但是，使用空白可极大提高脚本的可读性。

例如，以下面存在问题的 MEL 脚本为例：

int 			$scale = 0; 
string 
$text; if (rand(10 ) <
 1){ $scale = 
10; $text = "Exceptional scaling";} else $text = 
"Default scaling";

从技术上讲，上面的示例是正确的。但是，脚本格式导致其阅读困难。可以使用空格、制表符和空行来使脚本更易于理解。以下是更好地使用了空白的同一脚本：

int $scale = 0;
string $text;
if (rand(10) < 1) {
 $scale = 10;
 $text = "Exceptional scaling";
}
else 
 $text = "Default scaling";
