在自定义或网络位置设置 Bifrost

要从自定义位置（例如网络上）安装并运行适用于 Maya 的 Bifrost，请设置 MAYA_MODULE_PATH 环境变量。如果并行安装了多个版本，也可以使用 MAYA_MODULE_PATH 指定要使用的 Bifrost 插件的本地版本。

  1. 将安装目录（例如 ../Bifrost/Maya2023/2.7.0.1）复制到自定义位置（例如 /my/shared_path/Maya2023/2.7.0.1），路径不能包含下列任何字符：

    : * ? " < > |

  2. 将 MAYA_MODULE_PATH 环境变量设置为包含该目录（在本示例中为 /my/shared_path/Maya2023/2.7.1.0）。注意：

    • 变量应指向包含 .mod 文件的文件夹，而不是指向 .mod 文件本身。.mod 文件使用相对路径，无需修改。
    • 在 Linux 和 MacOS 上，格式 //*/... 不起作用。请改为使用 //... 或 /.//*/...。
    • 将使用在 MAYA_MODULE_PATH 中找到的第一个 Bifrost 版本。

  3. 启动 Maya。

通常，您应该将 Bifrost 安装到 Maya 安装目录之外的位置。请勿将 Bifrost 安装到 Maya 安装的插件目录内的某个位置。否则，您可能会遇到以下错误：getFullyQualifiedNameEv、Amino11NamedEntity7getNameEv 和 AminoNamedEntity。此外，以下是您可能在 Linux 上看到的错误示例： 

../lib/libBifrostBoardExec_1_0.so: undefined symbol: _ZNK5Amino11NamedEntity7getNameEv (bifrostGraph) // 

../scripts/others/pluginWin.mel line 290: Unable to dynamically load : .../bifrost/plug-ins/bifrostshellnode.so

../lib/libbifrostmayaconversion_1_0.so: undefined symbol: _ZNK5Amino11NamedEntity21getFullyQualifiedNameEv (bifrostshellnode) //
注意：

不支持从自定义位置卸载。您需要手动移除安装目录中的文件。
