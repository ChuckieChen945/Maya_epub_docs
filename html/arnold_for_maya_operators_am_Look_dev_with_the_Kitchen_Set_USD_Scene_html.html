使用厨房集 USD 场景进行外观制作

使用 Arnold 渲染的 USD 厨房集资源

在本教程中，我们将介绍如何从 Pixar USD 下载页面打开 kitchen_set.USD 文件。其中一种方法是使用 Arnold 程序 （MtoA 的替代对象）。这是可以单独加载 Arnold 的代理节点，但我们也可以直接加载 USD 文件。我们将介绍如何应用不同的外观和着色器，以及如何使用运算符更改 USD 文件的内容。

要下载对应的 kitchen_set.USD 文件，请单击此处。
注意：

有关 USD 的详细信息，请单击此处。

加载 USD 文件

  • 创建 Arnold 替代对象。我们将使用此方式打开 kitchen_set.USD 文件。

  • 在程序/替代对象的路径中打开 kitchen_set.USD 文件。

  • 将“视口绘制模式”(Viewport Draw Mode)从“边界框”(Bounding Box)（默认值）更改为“着色多边形线”(Shaded Polywire)，以便可以在视口中查看厨房的内容。
  • 在 X 轴方向将其旋转到 -90 度，使其正立。

  • 创建 *skydome_light（“Arnold -> 灯光 -> 天穹灯光”(Arnold -> Lights -> Skydome Light)）。将采样数*增加到 3。
  • 在 Arnold 渲染视图窗口中渲染场景。

如果我们查看替代对象的“文件内容”(File Contents)，可以看到 kitchen_set USD 文件的内容。您可以在 Arnold 渲染视图窗口中选择对象，并在视口和“文件内容”(File Contents)列表中查看所选内容。

覆盖

添加着色器指定

我们对 USD 文件的内容创建一些覆盖。

  • 将着色器指定添加到“文件内容”(File Contents)窗口中的“Kitchen_Table1 -> 几何体”(Kitchen_Table1 -> Geom)。

  • 选择右侧的着色器，然后选择一个 standard_surface 着色器。将其 base_color 更改为红色。这时效果应类似以下渲染。

几何体指定

除了着色器覆盖，我们还可以覆盖网格的所有 Arnold 属性（如细分）。例如，我们可以看到，墙上的铸铁几何体 (/Kitchen_set/Props_grp/North_grp/NorthWall_grp/CastIron_1/Geom/pCylinder151) 具有较少的细分数。

在渲染视图窗口中选择 CastIron/Geom/pCylinder151

  • 选择 pCylinder151 几何体后，选择“添加赋值 -> 多边形网格 -> subdiv_type”(Add Assignment -> polymesh -> subdiv_type)和 subdiv_iterations（设置为 2）。

现在，您应该注意到，该几何体是平滑的，不像以前未使用多边形网格覆盖时那样是面状的。

运算符

使用 Disable 隐藏几何体

我们还可以应用 Arnold 运算符来隐藏几何体。

  • 在“过滤器”(Filter)*中键入 *Fridge 以显示冰箱几何体。
  • 选择“冰箱”(Fridge)组：/Kitchen_set/Props_grp/North_grp/FridgeArea_grp
  • 从“添加运算符”(Add Operator)列表中选择 *disable*。您应该会注意到，渲染中不再显示冰箱。

使用 Disable 运算符隐藏冰箱组

使用 set_transform 移动对象

我们还可以使用 Arnold 运算符移动对象。

  • 选择 ChairB_1 组，然后对其应用 set_transform 运算符。

  • 选择 set_transform 右侧的箭头，然后在属性编辑器中更改 ChairB_1 组的变换。

使用 aiSetTransform 运算符转换椅子

显示图形(Show Graph)

  • 我们还可以可视化所创建的运算符节点，并可以使用“显示图形”(Show Graph)按钮查看附加到替代对象的这些节点。

运算符 -> 替代对象

外观

现在，我们可以创建多个版本或“外观”，以更改可用于外观制作目的的外观。

  • 创建一个新外观（+ 图标），并将其命名为“Props”。

  • 选择 Arch_grp，然后创建 disable 运算符。现在，您应该能看到 Arnold 渲染视图窗口中渲染的 Props。
“Props”外观可见，其他几何体已使用运算符进行禁用（隐藏） “默认”外观

导入/导出运算符

可以在资源、镜头甚至应用程序之间传递运算符。在照明阶段，您可能想要进行一些最后的更改，此时可以将其作为覆盖应用于场景中。

  • 选择替代对象后，转到“Arnold -> 运算符 -> 导出运算符图表”(Arnold -> Operators -> Export Operator Graph)。确保“导出指定的着色器”(Export Assigned Shaders)处于启用状态，以便导出 USD 中新创建的所有着色器。

现在，我们将运算符导入到新的 USD 厨房场景后：“Arnold -> 运算符 -> 导入运算符图表”(Arnold -> Operators -> Import Operator Graph)，您应该会在“节点编辑器”(Node Editor)窗口中看到这些运算符。您只需连接替代对象的 lookSwitch -> 运算符。

导入的运算符连接到替代对象

希望您能够了解在 Arnold 中渲染时运算符和 USD 提供的强大功能。通过此工作流，您在应用程序之间切换时可以获得相同的外观，同时保留完全相同的渲染。
