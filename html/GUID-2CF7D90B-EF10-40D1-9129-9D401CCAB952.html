插件管理器

  • 在主菜单栏中，选择“窗口 > 设置/首选项 > 插件管理器”(Windows > Settings/Preferences > Plug-in Manager)。

“插件管理器”(Plug-in Manager)可以标识加载到 Maya 的插件。如果经常使用某一插件，则可以确保其始终存在。“插件管理器”(Plug-in Manager)会自动扫描插件路径中的所有目录，并列出可用的插件功能。

插件是用于扩展 Maya 功能的模块。文件转换器是用于导入和导出各种文件格式的插件。可创建或购买专用插件，以便针对特定作业对 Maya 进行自定义。

通过添加插件可实现下列功能：

  • 文件转换器
  • 工具
  • 对象（节点）
  • MEL 命令
  • 设备驱动程序

请参见 Maya 随附的插件列表，以及就缺少插件的疑难解答。

搜索

在“搜索”(Search)字段中输入文本，插件结果将自动过滤以匹配键入的文本。

“应用于全部”(Apply To All)
  • 当“应用于全部/已加载”(Apply To All/Loaded)框处于选中状态时，会批量加载当前会话的所有插件。
  • 当“应用于全部/自动加载”(Apply To All/Auto load)框处于选中状态时，所有的插件在下次启动 Maya 时会自动批量加载。

取消选中“应用于全部”(Apply To All)框将批量卸载插件。

注： 每个插件目录都有其自己的“应用于全部”(Apply To All)；“应用于全部”(Apply To All)将仅应用于当前插件目录中的插件。如果您想加载所有插件，则必须选中每个插件目录的“应用于全部”(Apply To All)。
已加载

启用时，插件针对当前的 Maya 会话加载。

自动加载

启用时，您下次启动 Maya 时插件会自动加载。

信息图标

若要查看有关特定插件的信息，请加载该插件，然后单击插件旁边的 i 图标来打开“插件信息”(Plug-in Information)窗口。

“插件信息”(Plug-in Information)窗口

在“插件管理器”(Plug-In Manager)中单击插件名称旁边的 图标时，此窗口将会打开。Maya 会缓存当前插件列表。如果添加了新的插件，请单击“刷新”(Refresh)以更新任何更改。

系统将会针对选定的插件显示以下信息：

名称

插件的名称。在 Linux 上，插件的扩展名为 .so。在 Windows 上，插件的扩展名为 .mll。在 Mac OS X 上，扩展名为 .bundle。

路径

文件的位置。

  • 在 Linux 上，默认插件位置为：/usr/autodesk/maya<版本>/bin/plug-ins
  • 在 Windows 上，默认插件位置为：drive:\Program Files\Autodesk\Maya<版本>\bin\plug-ins
  • 在 Mac OS X 上，默认位置为：/Applications/Autodesk/maya<版本>/Maya.app/Contents/MacOS/plug-ins 和 /Users/Shared/Autodesk/maya/<版本>

    Maya 会对这两个位置都进行检查，寻找有效插件。

供应商

插件的制造商。

插件版本(Plug-in Version)

插件的版本号。

适用于 API 版本(For API Version)

插件编译所针对的 Maya API（应用程序编程人员界面）版本。

注：

如果要加载插件的 Maya 版本在日期上早于或晚于编译插件时所面向的目标 Maya 版本，则无法加载该插件。也就是说，如果插件是为 Maya 2020 所创建，则该插件在 Maya 2022 内无法使用。

自动加载(Auto Load)

表示插件是否已标记为自动加载。

已加载(Is Loaded)

表示插件是否已加载。

插件功能(Plug-in Features)

显示由该插件添加的功能列表（例如，命令、依存关系节点、文件转换器）。

若要显示其他插件，请单击三角形，打开“注册的其他插件”(Other Registered Plug-in)部分。
