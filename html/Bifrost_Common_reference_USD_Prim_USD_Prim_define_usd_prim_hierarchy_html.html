define_usd_prim_hierarchy

创建 Bifrost-USD 基本体定义的层次。将输出连接到 add_to_stage 节点下游时，会在后台文件中创建 USD 基本体层次。

例如，将“路径”(path)设置为 /model/geo/grp1/mesh，并将“类型”(types)设置为 Xform Scope Xform Mesh 时，将创建以下 USD 层次：

 `--model [def Xform] (kind = group)
     `--geo [def Scope] (kind = group)
         `--grp1 [def Xform] (kind = group)
             `--mesh [def Mesh] (kind = group)

输入

层次(Hierarchy)

path

基本体层次的完整路径，例如 /Car/geo/doors/left。此处输入的路径将添加到任何已连接子对象指定的路径前面。同样，如果此节点的输出连接到基本体定义下游的 children 端口，则此路径将被附加到该处指定路径的后面。

parent_is_scope

即使 types 输入设置了其他类型，也要确保层次的父对象为“范围”(Scope)。当层次是由脚本从没有此类“范围”(Scope)类型的主机应用程序转换层次而创建时，这将非常有用。

types

以空格分隔的 USD 基本体类型的字符串。

default_type

如果基本体类型数小于基本体数，则 default_type 将用于设置其余基本体的类型。

网格(Mesh)

normal_per_vertex

按顶点存储法线（如果存在于 Bifrost 网格上）。

subdivision_scheme

网格的细分方法。启用 normal_per_vertex 时，细分方法将被忽略。

use_frame

是否在指定的 frame 处设置几何体数据。

frame

启用 use_frame 时要为其设置几何体数据的帧。

attribute_definitions

（可选）连接叶网格基本体的一个或多个属性定义。可以在端口上单击鼠标右键，然后选择“连接节点”(Connect Node)以快速添加和连接 define_usd_attribute 节点或常用预设（如 define_usd_transform 或 define_usd_display_color）。

relationship_definitions

（可选）连接叶网格基本体的一个或多个关系定义。可以在端口上单击鼠标右键，然后选择“连接节点”(Connect Node)以快速添加和连接 define_usd_relationship 节点。

leaf_mesh

如果连接了 Bifrost 网格，则将层次的最后一个基本体定义创建为相应的 USD 网格。用“网格”(Mesh)类型覆盖 types 输入的最后一个类型。
