标准路径

当 Maya 将环境变量设置加载到其内存时，它会将一些标准路径添加到某些环境变量以确保 Maya 需要运行的某些内容始终可用。这将影响下列变量： 

MAYA_PLUG_IN_PATH 
MAYA_MODULE_PATH 
MAYA_SCRIPT_PATH 
XBMLANGPATH

当您亲自设定这些路径时，使用操作系统方法设置的变量将覆盖所有其他设置，而在 Maya.env 中设置的变量将覆盖标准设置；如果标准设置没有被系统变量或 Maya.env 覆盖，将使用标准设置。

由于您的路径设置优先于标准 Maya 路径，因此当您的路径设置中的一个目录包含与 Maya 需要运行的文件同名的文件时，可能会覆盖这些文件。应注意避免这种情况，因为这可能会导致 Maya 无法正确执行。

在 Windows 和 Linux 中，您可以通过在命令行中键入 maya -default 来尝试启动仅具有默认设置的 Maya。

其他路径设置

对于系统 PATH 变量，Maya 将添加 $MAYA_LOCATION/bin。在 Linux 中，它还将添加 /usr/autodesk/com/bin。

Maya 将检查由 HOME 变量（Linux、Mac OS X）或 USERPROFILE 变量 (Windows) 指定的目录是否存在和可写。如果不存在或不可写，Maya 将提示您提供可写主目录。请注意，HOME 和 USERPROFILE 不能在 Maya.env 文件中设定。
