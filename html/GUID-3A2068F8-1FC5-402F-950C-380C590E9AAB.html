编解码器支持

“摄影机序列器”(Camera Sequencer)和“播放预览”(Playblast)选项使用视频编解码器以将摄影机快照组合成影片片段进行播放。Maya 中不包含用于播放预览的特定编解码器，而是依靠系统中已安装的编解码器。请注意，并非所有的编解码器都与 Maya 兼容。

下表列出了每个平台上的受支持影片类型以及已知问题。

平台 Windows（64 位） Mac OS X Linux

已知问题：

不支持 Motion JPEG 和 RGB 565 16 位颜色。

不支持为 DirectPlay 或 MediaFoundation 开发的编解码器插件。

性能在 64 位系统上不稳定。

 

如果在播放预览时遇到困难，可能需要下载并安装附加的编解码器。由于可用编解码器的数量和种类繁多，Autodesk 不提供限定编解码器的列表。

作为一般规则，请确保要在其中查看播放预览影片的计算机与用于创建影片的计算机包含的编解码器相同。例如，如果在 Linux 系统上播放预览影片并且希望在 Windows 或 Mac OS X 系统上观看影片，请确保 Windows 或 Mac OS X 系统上支持且安装了相同的编解码器。

要查看系统上的可用编解码器的列表，请打开任意播放预览选项（位于“摄影机序列器”(Camera Sequencer)的“播放预览”(Playblast)菜单下或位于“播放预览选项”(Playblast Options)窗口中）。选择要输出的影片的“格式”(Format)，然后查看“编码”(Encoding)下拉列表中的可用编解码器的列表。在安装附加编解码器时，系统会将它们添加到该列表中，且在下次运行 Maya 时即可使用它们。

注：

安装的编解码器会出现在“编码”(Encoding)列表中，即使它们与 Maya 不兼容也是如此。

请参考下面有关每个平台的编解码器支持的其他注释和异常的章节。

Mac OS X 编解码器注释

64 位 Mac 系统上的可用编解码器的列表不会发生更改，即使您安装附加编解码器也是如此。如果已安装用于播放预览的附加编解码器，则必须在 32 位模式下运行 Maya 以访问这些编解码器。

Linux 编解码器注释

Maya 当前使用 libquicktime（版本 1.1.4）且包含以下编解码器：png、jpeg、mjpa、raw、v308、v408、v410、yuv2、yuv4、yv12、2vuy、v210。如果要尝试其他编解码器，请它们安装在 /maya/bin 目录中。

如果计划在 Linux 系统上输出影片并在 Windows 或 Mac OS X 系统上观看影片，请使用 png、raw、jpeg、mjpa、v408、yuv2 或 2vuy 获得最佳效果。
