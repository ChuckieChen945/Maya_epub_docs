2D 运动模糊问题疑难解答

仅适用于 Maya 软件渲染。

如果对象周围显示暗色光晕

请尝试以下操作：

  • 确保背景颜色为黑色
  • 如果场景中有雾，确保打开“平滑 > 颜色”(Smooth > Color)。

连同背景一起移动透明对象

在这种情况下，背景也会模糊，即使它不不应该模糊。解决方法是先单独模糊透明对象，然后再与场景的其余部分进行合成。对于包含大量透明度的复杂场景或透明粒子，实施此解决方法会很难。

移动对象后面的详细背景

一些详细信息可能会丢失，因为必须做出与移动对象遮挡的背景区域有关的假设。解决方法是模糊移动对象（不包括背景），然后合成结果。

旋转对象

可能看起来不太正常，因为必须做出与这些对象的后侧应看似什么有关的假设。尝试使用 3D 运动模糊。

对象从图像外部进入或离开图像

框架的边可能不会获得正确的详细信息，因为必须做出与刚好位于框架外面的对象颜色有关的假设。解决方法是渲染能够覆盖原始图像的较大图像，然后将其裁剪至所需大小。

3D 和 2D 的渲染结果有很大不同

不要尝试混合通过两种不同模糊操作进行渲染的图像。

使用 IPR 调试 2D 运动模糊

当使用 IPR 调试 2D 运动模糊时，自动更新无法始终正常完成。调整材质或灯光的属性以强制执行正确更新，或框选调整区域以强制执行更新。

2D 运动模糊瑕疵

当在包含雾的场景或透明对象前面有实体对象的场景中进行渲染时，或当背景颜色不是黑色时，2D 运动模糊将导致出现瑕疵。

有以下三种可能的解决方法：

  • 从场景中移出透明对象、雾、辉光和背景颜色。渲染 2D 模糊，然后将这些元素重新合成到该场景中。
  • 将“平滑值”(Smooth Value)设定为 0。在命令行中，为 -m 0。对应的解决方法是跳过平滑屏蔽操作。结果，图像可能看上去会出现更严重的锯齿化现象。
  • 打开“平滑 > 颜色”(Smooth > Color)。在命令行中，为 -r 1。对应的解决方法是使用不同的平滑算法。结果，图像可能看起来更模糊。
