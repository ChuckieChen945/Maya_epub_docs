cacheFile 节点属性

该节点存储单个 nDynamic 缓存 (nCache) 和流体效果 nCache 的所有设置。该节点属性的更改不会影响它在磁盘中的关联缓存文件。请参见 cacheFile。

在依存关系图和“属性编辑器”(Attribute Editor)中，nCache 表示为 nNodeName#Cache# 节点。例如，windTunnelShape1Cache1、windTunnelShape1Cache2 和 windTunnelShape1Cache3 是三个 nCache 节点的名称，这三个节点均包含名为 windTunnel 的流体效果的缓存设置。在“Trax 编辑器”(Trax Editor)中，nCache 表示为 nCache 片段。更改 nCache 节点的设置时，也会更改其对应的 nCache 片段的设置。

缓存文件

基础目录(Base Directory)

指定当前 nCache 节点的原始缓存文件在磁盘中的目录。默认情况下，基础目录是：\maya\projects\default\data。

缓存名称(Cache Name)

指定当前 nCache 节点的原始缓存文件在磁盘中的名称。

启用(Enable)

启用时，其流体对象在播放时将使用当前 nCache 节点的模拟。禁用时，其流体对象在播放时不会使用当前 nCache 节点的模拟。

开始帧(Start Frame)

指定当前 nCache 节点的播放开始时间。可以更改该属性，以修改 nCache 节点开始播放的时间。

缩放(Scale)

展开或缩小当前节点的 nCache 的时间范围。如果增加比例，则节点的 nCache 模拟播放速度将减慢。减小比例会有相反的效果。

保持(Hold)

指定帧数以保持当前节点的 nCache 的上一帧。

前方循环(Pre Cycle) / 后方循环(Post Cycle)

指定位于当前节点的 nCache 之前（“前方循环”(Pre Cycle)）和之后（“后方循环”(Post Cycle)）的循环数。

可以使用循环来重复 nCache 中包含的模拟。

源开始(Source Start) / 源结束(Source End)

对于当前节点的 nCache 包含的原始缓存，指定其帧范围在磁盘中的开始帧编号和结束帧编号。如果当前节点的 nCache 已修剪，则“源开始”(Source Start)和“源结束”(Source End)值将与“原始开始”(Original Start)或“原始结束”(Original End)值不匹配。

原始开始(Original Start) / 原始结束(Original End)

指定当前节点的原始 nCache 文件在磁盘中的开始帧编号和结束帧编号。

反向(Reverse)

启用时，当前节点的 nCache 将从缓存的上一帧开始反转播放。默认情况下，“反转”(Reverse)处于禁用状态。

往返(Oscillate)

启用时，当前节点的 nCache 将在反转播放和向前播放之间往返其他每一帧，从而创建连续播放循环。默认情况下，“往返”(Oscillate)已禁用。

多线程(Multi Thread)

启用时，将在播放期间为多线程设置当前节点的 nCache。默认情况下，“多线程”(Multi Thread)已禁用。

队列大小(帧)(Queue Size (frames))

指示在启用多线程、但并未经过设置以预加载整个缓存的情况下，Maya 尝试预加载并保持在内存中的 nCache 节点缓存的帧数。默认情况下，“队列大小(帧)”(Queue Size (frames))为 20。

显示加载进度(Display Load Progress)

启用时，将显示已满的多线程缓存内存队列所占的百分比。默认情况下，“显示加载进度”(Display Load Progress)已启用。

缓存描述(Cache Description)

显示一些包含在当前 nCache 节点的 XML 描述文件中的信息。有关“缓存描述”中所含值的详细信息，请参见 nCache XML 描述文件。
