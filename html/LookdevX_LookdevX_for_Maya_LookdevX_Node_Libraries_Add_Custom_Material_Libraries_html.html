Add Custom Material Libraries

In LookdevX, you can add custom material libraries and access them in the Viewport contextual menu under "Assign New/Existing Material".

To add a custom material library, you need to register one of the following MEL callbacks:

  • addRMBNewMaterialMenuItems
  • addRMBExistingMaterialMenuItems

These callbacks can be registered by any client using a MEL script. Here is how you can do it:

  • Use the callbacks command to add your custom callback. For example: callbacks -addCallback "myCustomCallback" -hook "addRMBNewMaterialMenuItems" -owner "ClientPluginName";

  • Define your custom callback as a global procedure. This procedure will be called when the callback you registered is triggered. For example:

    global proc myCustomCallback(string $object)
{
    // Add your custom 'menuItem' here
}

Set up custom definitions

MaterialX

To set up a custom definition of nodedefs in MaterialX that can be loaded into LookdevX dynamically or at startup use the Environment variable MATERIALX_SEARCH_PATH then enter a path pointing to where the desired library is located.

USD

To set up a custom definition of nodedefs in USD that can be loaded into LookdevX dynamically or at startup use the Environment variable PXR_MTLX_STDLIB_SEARCH_PATHS then enter a path pointing to where the desired library is located.

Arnold

To set up a custom definition of nodedefs in Arnold that can be loaded into LookdevX dynamically or at startup, use the Environment variable ARNOLD_MATERIALX_NODE_DEFINITIONS, then enter a path pointing to where the node definitions are located.
