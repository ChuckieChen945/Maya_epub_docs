orient_points_by_geometry

根据对网格几何体进行采样的情况，设置基于点的对象的 point_orientation 特性。方向的 Y 轴与几何体的法线对齐，其他轴与其相切平面对齐。例如，要对齐散射在景观网格上的实例时，这很有用。

输入

points

要确定方向的点。

locations

要对 geometry 输入的法线和切线进行采样的位置。例如，这可以是几何体查询节点的输出。

geometry

要在给定位置采样的网格几何体。

normal_mode

要用作 Y 轴的法线：point_normal 或 face_vertex_normal。

tangent_mode

如何确定曲面切线：

  • Connectivity 表示根据基础网格拓扑的边流计算切线。
  • UV 表示根据网格的 point_UV0 特性计算切线。

prioritize_normal

控制法线和切线不正交时发生的情况。当 prioritize_normal 为 true 时，生成的方向的 Y 轴与法线完全对齐。当此项为 false 时，优先将 X 和 Z 轴与相切平面对齐。

normal_override

（可选）用于提供要使用的具体法线。连接了有效数据时，将使用这些线，而不是网格拓扑。这是一个解释的自动端口，其接受以下类型：

  • 连接单个 float3 以将其值用于所有点。
  • 连接 array<float3> 以对每个点使用数组的相应元素。可根据需要调整数组大小以匹配点数。
  • 连接 string 以使用具有相应名称的特性给定的向量（每个点的 float3）。
  • 连接 VectorField 以使用点的当前位置处的场的值。

tangent_override

（可选）用于提供要使用的具体切线。连接了有效数据时，将使用这些线，而不是网格拓扑。这是一个解释的自动端口，其接受与 normal_override 相同的类型。

rotate_existing_values

当此选项为 true 时，将使用从网格几何体计算的方向来旋转点对象的现有 point_orientation 值。当此选项为 false 时，将替换现有值。

blend_weights

控制要应用于每个点的方向量。权重为 0 表示点根本不受影响，权重为 1 表示点的方向完全受控制。

如果未连接任何对象，则所有点的默认权重为 1。这是一个解释的自动端口，其接受以下类型：

  • 单个 float 解释为应用于所有点的常量值。
  • array<float> 解释为每个点的值。可根据需要调整数组大小以匹配点数。
  • array<bool> 被解释为每点的 0 (false) 或 1 (true) 值。
  • string 解释为现有几何特性的名称，并使用其值。
  • ScalarField 将在几何体组件的每个位置处进行采样。
  • array<long> 指定要为其指定值 1 的索引。对于其他索引，则指定值 0。

输出

out_points

具有修改后的 point_orientation 值的点对象。

point_orientation

修改后的 point_orientation 数组。
