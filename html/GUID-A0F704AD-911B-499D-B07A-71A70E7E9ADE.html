Linux 特定变量

MAYA_WORKSPACES_ALWAYS_SHOW_TAB
将窗口或工具拖动到其容器之外时，保留标签/夹点控制柄。这样，在某些 Linux 系统（Centos 7.4.1708 和 KDE 4.14.8）上将窗口取消停靠后，可以使用标签（例如，使用属性编辑器或工具箱）或夹点控制柄（例如，使用时间滑块或命令行）重新停靠窗口，而以前不允许这样操作。

将环境变量设置为 1 以正常停靠和取消停靠窗口。

MAYA_DEBUG_NO_SIGINT_HANDLING

在 Linux 平台上运行 Maya 单机版时，如果 Maya 在您尝试使用 kill -2 或 kill -2 python-bin 结束 Python 进程时以 0 状态退出，则可以设置环境变量 MAYA_DEBUG_NO_SIGINT_HANDLING = 1，使 Maya 以非 0 状态退出。

__GL_MAYA_OPTIMIZE

NVIDIA® Linux 驱动程序 295.59 禁用了某些 OpenGL 优化，可能会影响 Maya 性能。如果出现性能降低的情况，请将此变量设置为 1，以启用优化。若要解决 NVIDIA 优化问题，将此变量设定为 0 以禁用优化。默认情况下，此变量处于禁用状态。

MAYA_MMSET_DEFAULT_XCURSOR

创建在使用“标记菜单”(Marking Menus)后重置光标时要使用的左指针光标。这在使用 GNOME 窗口管理器时很有用，否则这些窗口管理器会恢复到默认的“X”光标。

MAYA_SOUND_SCRUB

禁用擦除声音功能。

MAYA_SET_PARENT_HINT

设定为 0 将禁用建立窗口的父子关系。

MAYA_STYLUS_DEVICE

如果已经自定义了光笔的名称，可以将该环境变量设定为与 Maya 的光笔同名，以方便识别设备。

MAYA_SET_XERROR

允许用户强制重新安装 XError 处理程序。

MAYA_FORCE_SHOW_ACTIVATE

在使用 Gnome 桌面（窗口边框窗口管理器）时，使用该环境变量有助于确保弹出窗口提升到列表顶部。

MAYA_ACTIVE_STEREO_REFRESH

在启用主动立体时，以及已关闭热盒/菜单后，使用该环境变量可强制视口刷新。

XLIB_SKIP_ARGB_VISUALS

当应用程序请求使用可能的视觉效果时，排除 ARGB 视觉效果在考虑范围外。当 X 桌面合成扩展处于启用状态时，该变量很有用。

MAYA_TTIP_OFFSETX

允许 Maya 工具提示在水平（X 轴）方向按给定像素数偏移。

MAYA_TTIP_OFFSETY

允许 Maya 工具提示在垂直（Y 轴）方向按给定像素数偏移。
