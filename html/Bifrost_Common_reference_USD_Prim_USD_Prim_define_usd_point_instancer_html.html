define_usd_point_instancer

基于 Bifrost 点对象定义 USD 点实例化器基本体。将输出连接到 add_to_stage 节点下游时，这会在后台文件中创建实例。此节点是基于 define_usd_prim 的专用复合。

输入

instancer_path

新实例化器基本体的路径，例如：/rocks 或 /debris/rocks。如果此节点的输出连接到基本体定义下游的 children 端口，则此路径将被附加到该处指定路径的后面。

points

连接 Bifrost 点对象。

prototype_definitions

连接要实例化的基本体的基本体定义。可以在端口上单击鼠标右键，然后选择“连接节点”(Connect Node)以快速添加和连接任一常用节点。

选择

selection_mode

控制如何选择原型。连接 instance_id_override 时，该项将被忽略。

  • None：原型未被实例化。
  • Sequential：第一个连接的原型针对点 0 实例化，第二个针对点 1 实例化，依此类推。选择为环绕式，如果有 N 个连接，则第一个连接的原型将用于点 N，依此类推。
  • Random：为每个点随机指定原型。可使用 probability_curve 和 seed 调整分布。

instance_id_override

直接控制原型的实例化方式。用户可以连接与点数匹配的整数数组，其中每个元素指定对应点使用的原型。

probability_curve

selection_mode 为 Random 时，编辑曲线以调整分布。

seed

selection_mode 为 Random 时随机数生成器使用的种子。

默认材质

create_default_material

为实例指定默认材质。

diffuse_color

默认材质的漫反射颜色。

texture_file_path

要使用的纹理文件的路径，而不是 diffuse_color。

时间

frame、use_frame

使用 frame 和 use_frame 可以在点实例化器上设置当前时间，这将影响从其获取实例化几何体的时间采样。

变量选择

这些参数是可选的。

variant_set_name

输入基本体定义变量的变量集的名称（可选）。如果保留为空，则基本体不是任何变量集的成员。

variant_name

此基本体定义的变量的名称。

输出

point_instancer_definition

新基本体定义。可以将其连接到 add_to_stage 节点的 prim_definitions 端口，或连接到其他 define_usd_prim 或类似节点的 children 端口。
