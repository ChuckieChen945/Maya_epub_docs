Linux 版 Maya 疑难解答

以下部分介绍了 Linux 版 Maya 独有的一些问题，并提供了建议解决方法。

缺少依存关系

如果收到有关 Maya 因缺少依存关系而无法启动的消息，请参见 Maya 所需的其他 Linux 库相关部分，并安装系统中不存在的任何库。

捕捉阻滞

按下任意键启用捕捉（例如，V、C 或 X）时，捕捉可能会在释放热键后仍处于活动状态。

解决方法

在未按下热键的情况下再次单击鼠标中键。

选择 NURBS 曲面的等参线时出现显示问题

选择 NURBS 曲面的等参线时可能会显示奇怪的图案。

解决方法

再次选择等参线即可移除此图案。

将菜单项或选项窗口放置到工具架上

在 Maya 中发布菜单，然后按 Shift+Ctrl+Alt 并单击一个菜单项或选项窗口，将其放置到工具架上时，什么都未发生。

解决方法

改用 Shift+Alt-鼠标右键。

无法选择菜单项

单击扩展选项卡面板以在选项卡面板上显示弹出菜单（就像在“属性编辑器”(Attribute Editor)中一样）时，光标显示在菜单左边缘内侧，但没有菜单项亮显。实际已选择光标下的项目，但该项目未亮显。

解决方法

将光标稍微向左移动，取消选择菜单，然后返回菜单，选择菜单项。

选项子菜单非始终可见

有时文件放置对话框不以可读或可视形式显示选项子菜单。

解决方法

尝试手动调整对话框大小以便完全显示选项子菜单。

Linux 和 GNOME

以下是一些配合 Maya 使用 Linux 和 GNOME 时可能会遇到的问题。

鼠标光标显示为黑色“X”

有时单击鼠标右键，鼠标指针会变成黑色的“X”，这种情况在 Maya 会话中存在，甚至在重新启动 Maya 后也依然存在。

解决方法

设置环境变量 MAYA_MMSET_DEFAULT_XCURSOR；值无关紧要。设置此变量后，Maya 会在退出“标记菜单”(Marking Menu)/“热盒”(Hotbox)后将光标设置为左指向箭头光标。

使用了此解决方法后再退出热盒，光标可能会保持为右指向红色箭头。若要解决此问题，请在退出热盒后在场景上单击鼠标右键。光标将变为默认的左指向黑色箭头。但是，在进入热盒后红色箭头不会再显示，直到退出并再次登录。

标记菜单/热盒

在 Linux 上使用 GNOME 窗口管理器时，退出“标记菜单”(Marking Menu)/“热盒”(Hotbox)后，光标可能会重置为默认的桌面“X”光标。

解决方法

设置环境变量 MAYA_MMSET_DEFAULT_XCURSOR；值无关紧要。设置此变量后，Maya 会在退出“标记菜单”(Marking Menu)/“热盒”(Hotbox)后将光标设置为左指向箭头光标。

使用了此解决方法后再退出热盒，光标可能会保持为右指向红色箭头。若要解决此问题，请在退出热盒后在场景上单击鼠标右键。光标将变为默认的左指向黑色箭头。但是，在进入热盒后红色箭头不会再显示，直到退出并再次登录。
