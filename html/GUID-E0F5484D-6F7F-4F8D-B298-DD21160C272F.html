集合表示

集合表示是可以在场景内或场景外（具体取决于当前任务的要求）交换的生产资产的替换数据。

可以在执行快照序列、场景布局或任何其他需要通过场景完成的工作时快速替换最适合的生产资产版本。例如，如果使用精细的表设置布局场景，则可以使用简单定位器或缓存对象来表示盘子、刀具和其他餐具，并在工作时注意即时的性能改进。

共有三种表示类型：定位器、缓存和场景。每个生产资产都可以有每种类型的多重表示。一次只能有一个资产表示处于活动状态。（另请参见切换活动表示。）

提示： 您还可以将对象转化为边界框，并将其用作集合表示。请参见将边界框用作表示。
活动表示状态与场景一同保存。重新打开场景时，将自动加载上一个活动表示。
重要： 请勿在表示文件中使用名称空间。

定位器表示

定位器表示是标准的 Maya 定位器对象，用于标记空间中的点。定位器表示是默认的表示类型，并在生产资产处于活动状态时显示在其枢轴点处。

缓存表示

缓存表示是生产资产的基于 Alembic 的缓存版本，如 Maya GPU 缓存的对象。缓存表示为不可编辑但可快速加载的版本。必须先将对象保存到 Alembic 缓存，然后才能将其添加为缓存表示。

有关基于 Alembic 的缓存的详细信息，请参见 Alembic 缓存和“缓存”(Cache)菜单。

场景表示

场景表示是 .ma 或 .mb 场景文件，其中包含生产资产的可充分渲染几何体。当场景表示处于活动状态时，对象的层级将显示在“大纲视图”(Outliner)中。

注： 您可以使用 Maya 场景集合 API 创建自定义的集合表示。有关详细信息，请参见 Maya 开发人员帮助中的 MFnAssembly 类。
