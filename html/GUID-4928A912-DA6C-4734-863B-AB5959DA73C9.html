渲染变量

AW_JPEG_Q_FACTOR

该变量可用于指定 Maya 渲染出的 JPEG 文件的质量。有效值介于 1 到 100 之间，且 100 为最高质量。默认值为 100。

AW_JPEG_SUB_SAMPLING

可用于控制渲染图像的子采样的质量。

AW_JPEG_SUB_SAMPLING 环境变量的值为：

L1xL2,Ch1xCh2,Gr1xGr2

其中：

  • L1 和 L2：亮度组件的采样因子
  • Ch1 和 Ch2：色度组件的采样因子
  • Gr1 和 Gr2：灰度组件的采样因子

以下所列为该环境变量的一些可能设置：

  • 2x2,1x1,1x1（默认）
  • 2x1,1x1,1x1
  • 1x1,1x1,1x1（提供最高的视觉质量）

例如：

  • mel% putenv "AW_JPEG_SUB_SAMPLING" "1x1,1x1,1x1"

每个 L1xL2、Ch1xCh2 和 Gr1xGr2 组件的值可介于 1 和 4 之间，其中 1x1 为最高质量。通常，组件值越低，最终 JPEG 的质量就越好，因为较小的采样块有利于捕捉更好的高频颜色信息。

提示：

若要获得高视觉质量，还应当将环境变量 AW_JPEG_Q_FACTOR 设置为“100”。该变量控制图像的总体质量，可能将所有设置遮罩为 AW_JPEG_SUB_SAMPLING。

MAYA_COLOR_MGT_NO_LOGGING
将此值设置为任意值以阻止将颜色管理 (OpenColorIO) 消息输出到脚本编辑器或“输出”(Output)窗口（也称为“控制台”）。
MAYA_COLOR_MANAGEMENT_SYNCOLOR
将此项设置为任何值，以便在不支持 OpenColorIO v2 的渲染器中使用 Maya 的旧版颜色管理系统（称为 SynColor）。必须先在操作系统中或直接在命令行上设置此项，然后从同一 Shell 启动 Maya。如果在 Maya.env 文件中或通过在 Maya 的“脚本编辑器”(Script Editor)中使用 putenv 设置此项，则不起作用。请参见使用 Maya 的旧版颜色管理 (SynColor)。
MAYA_COLOR_MANAGEMENT_POLICY_FILE

将此变量设置为启动 Maya 时加载的策略（外部颜色管理首选项文件）的完整路径和文件名。如果 MAYA_COLOR_MANAGEMENT_POLICY_LOCK 设置为 0 或取消设置，您仍可修改颜色管理首选项并更新或导出设置。还可以加载不同的策略，甚至清除路径以卸载该策略。新设置将用于当前会话，但重新启动 Maya 时将再次应用环境变量指定的策略。请参见使用颜色管理策略（外部首选项文件）。

MAYA_COLOR_MANAGEMENT_POLICY_LOCK

当 MAYA_COLOR_MANAGEMENT_POLICY_FILE 设置为有效的外部颜色管理首选项文件时，将此变量设置为 1 可防止颜色管理首选项被修改（在当前场景中切换颜色管理缸显示或禁用颜色管理时除外）。将其设置为 0 或取消设置可解除锁定颜色管理首选项。

MAYA_ENABLE_DEPRECATED_VIEWPORT
重要： 将在将来的 Maya 版本 (2024+) 中弃用此变量。

可用于访问 Maya 旧版视口。请参见 Maya 旧版视口了解相关设置。

MAYA_NO_JITTER_FINAL_COLOR

如果要量化到 8 位颜色，则对颜色应用一些随机度或抖动。若要启用该选项，请将该值设置为 1。若要禁用该选项，请将该值设置为 0（零），或保留其未定义。

MAYA_RENDERER_RT_BACKGROUND_COLOR

如果将此变量设置为 1，则 Maya 将在计算反射光线和折射光线时包括摄像机背景。如果将此变量设置为 0（零）或保留为未定义，则 Maya 将忽略这些光线的背景色。

MAYA_HW_FILE_TEXTURE_RESOLUTION_OVERRIDE

启用此标志可强制烘焙文件纹理在文件纹理或实际文件纹理维度上为纹理分辨率属性的最大值。

MAYA_OFFSCREEN_HRB

将此标志设置为 1，以便在“硬件渲染缓冲区 > 渲染”(Hardware Render Buffer > Render)菜单中显示“渲染屏幕外”(Render Offscreen)选项。启用后，使用“硬件渲染缓冲区”(Hardware Render Buffer)渲染序列时将使用屏幕外缓冲区。单帧渲染将继续渲染到屏幕上的缓冲区中。在屏幕外渲染期间，在“硬件渲染缓冲区”(Hardware Render Buffer)窗口上移动窗口通常不会影响渲染的帧。

注：

“渲染屏幕外”(Render Offscreen)选项仅在 Linux 和 Mac OS X 中受支持，而在 Windows 中不受支持。

MAYA_EXR_LONGNAME

EXR 1.7 版支持 252 个字符的通道名称。限制为 252 而非 255 是为了节省用于区分各个通道（例如，.R 表示红色通道）最多 3 个字符的扩展名的空间。但是，目前只有少量外部应用程序（例如，Flame）支持 EXR 1.7 版，而且 31 个字符以上的嵌入通道名称可能会破坏与不支持此版本的外部应用程序间的兼容性。默认情况下，会将通道名称截断为 31 个字符以便向后兼容。将该选项设置为 1 以允许长通道名称，这样通道名称即不会被截断。

MAYA_TEXCOORD_ORIENTATION

将此环境变量设置为 OPENGL 或 DIRECTX 以分别定义 MAYA_TEXCOORD_ORIENTATION_OPENGL 或 MAYA_TEXCOORD_ORIENTATION_DIRECTX 宏。定义宏时会将其设置为 1。

设置此环境变量后，CGFX 插件编译 CGFX 着色器时，会通过将 MAYA_TEXCOORD_ORIENTATION_OPENGL 宏或MAYA_TEXCOORD_ORIENTATION_DIRECTX 宏设置为 1 来编译该着色器，并使用指定的方向（OpenGL 或 DirectX）加载纹理。有关详细信息，请参见 CgFX 着色器的纹理坐标系的方向。

MAYA_QUICKTIME_ENCODING_GAMMA

您可以通过设置环境变量 MAYA_QUICKTIME_ENCODING_GAMMA，将用于编码 QuickTime 播放预览的 Gamma 自定义为所需的 Gamma。如果未设置此环境变量，则默认值为 2.2。

MAYA_VP2_DEVICE_OVERRIDE

使用此环境变量可选择要使用其运行 Viewport 2.0 的渲染引擎。设置了此环境变量时，它将覆盖交互式和批处理会话的用户首选项（“窗口 > 设置/首选项 > 首选项 > 显示”(Window > Settings/Preferences > Preferences > Display)）（使用 -hw2 选项）。

按照下面所示设置此环境变量：

  • VirtualDeviceDx11 可在 DirectX 11 模式下渲染 Viewport 2.0
  • VirtualDeviceGL 可在“OpenGL - 旧版”(OpenGL - Legacy)模式下渲染 Viewport 2.0
  • VirtualDeviceGLCore 可在“OpenGL - 核心配置(精确)”(OpenGL - Core Profile (Strict))模式下渲染
  • VirtualDeviceGLCoreCompat 可在“OpenGL - 核心配置(兼容性)”(OpenGL - Core Profile (Compatibility))模式下渲染。

取消设置该环境变量可再次使用用户首选项来选择渲染引擎。

注：

通过 Windows 远程桌面访问 Maya 时，除非使用的是 Nvidia GeForce 或 Quadro 显卡，否则不支持此环境变量。Maya 会自动将 Viewport 2.0 渲染引擎切换到 DirectX 11 模式进行远程会话。render.exe 命令也使用 DirectX 11 模式。有关在 DirectX 11 模式下运行 Viewport 2.0 的详细信息，请参见将 DirectX 11 与 Viewport 2.0 结合使用。

如果您使用的是 Nvidia GeForce 显卡并想使用“OpenGL - 核心配置(精确)”(OpenGL - Core Profile (Strict))或“OpenGL - 核心配置(兼容性)”(OpenGL - Core Profile (Compatibility))模式，可以从 Nvidia 下载工具，以通过 GeForce 驱动程序 R440 或更高版本启用 OpenGL 应用程序的 Windows 远程桌面流。若要使用此工具，请设置环境变量 MAYA_ALLOW_OPENGL_REMOTE_SESSION=1。

如果您使用的是 Nvidia Quadro 显卡，则只需设置 MAYA_ALLOW_OPENGL_REMOTE_SESSION 环境变量，而不需要 Nvidia 工具。

MAYA_ALLOW_OPENGL_REMOTE_SESSION

若要在远程桌面上以“OpenGL - 核心配置(精确)”(OpenGL - Core Profile (Strict))或“OpenGL - 核心配置(兼容性)”(OpenGL - Core Profile (Compatibility))模式运行 Viewport 2.0，请将此环境变量设置为 1。Nvidia GeForce 和 Nvidia Quadro 显卡均支持此功能。

对于 Nvidia GeForce 显卡，需要从 Nvidia 下载工具，以通过 GeForce 驱动程序 R440 或更高版本启用 OpenGL 应用程序的 Windows 远程桌面流。

对于 Nvidia Quadro 显卡，只需设置此环境变量。

MAYA_VP2_USE_LINEAR_NC_SPACE_DEPTH_PRIORITY

默认情况下，Maya 使用一种算法来确定深度优先级（即，决定哪些 UI 元素绘制在其他元素顶部）。此算法在剪裁空间中呈线性缩放，并当近剪裁平面距离很小时可提供更精确的结果。可以通过将该环境变量设置为 1，还原到旧的 (NDC) 深度优先级算法。

MAYA_VP2_OFF_NON_PE_FRAGMENT

如果当图像平面紧跟在移动的对象之后时运动模糊不生效，可以通过将 MAYA_VP2_OFF_NON_PE_FRAGMENT 环境变量设置为 1 来解决此问题。设置此环境变量可将图像平面包括在所有场景后期效果的计算中，如屏幕空间环境光遮挡、景深以及运动模糊。

MAYA_FORCE_DX_DEBUG_DEVICE

此环境变量可用于在开发 Viewport 2.0 插件时调试显卡问题。您还必须安装 DirectX SDK 才能使用此功能。将 MAYA_FORCE_DX_DEBUG_DEVICE 设置为 1 后，在 Viewport 2.0 中使用 DirectX 模式，DirectX 设备创建时 D3D11_CREATE_DEVICE_DEBUG 标志处于启用状态。

MAYA_OBJECT_SORT_USING_BOUNDING_BOX

Maya 针对“对象排序”(Object Sorting)透明度选项（“渲染器 > Viewport 2.0”(Renderer > Viewport 2.0) > ）使用了一种算法，可以提供更精确的结果。若要还原为旧版 (Maya 2013)“对象排序”(Object Sorting)算法，请设置 MAYA_OBJECT_SORT_USING_BOUNDING_BOX = 1。

MAYA_GPUCACHE_VP2_OVERRIDE_API

此环境变量确定 gpuCache 插件用来驱动 Viewport 2.0 的 API。默认情况下，此变量设置为 MPxSubSceneOverride。这是唯一支持的值。

如果不设置此环境变量，将使用 MPxSubSceneOverride API。

此变量也可以设置为 MPxDrawOverride，但此选项仅作为说明如何使用 MPxDrawOverride API 的开发人员工具包示例提供。它不支持屏幕空间效果，如屏幕空间环境光遮挡、运动模糊、透明度排序和逐像素照明。

MAYA_FORCE_DX_WARP

将环境变量 MAYA_FORCE_DX_WARP 设置为 1 可始终使用 WARP 软件光栅化器。

首次使用渲染器时，会在输出窗口指示使用了 WARP。

此外，平视显示仪（指示当前渲染器）也指示正在使用 WARP。

WARP 表示 Windows 高级光栅化平台（Windows Advanced Rasterization Platform）。有关详细信息，请参考 Microsoft 的文档。

注： WARP 可能不完全支持硬件光栅化器支持的所有功能。这一点在 Windows 7 平台上比在 Windows 8 平台上更显著。目前 WARP 并不是合格的配置。
MAYA_OGS_GPU_MEMORY_LIMIT

当同时使用 Maya 和其他 3D 应用程序时，您可以将 Viewport 2.0 内存限制设置为较低的值，为其他 3D 程序预留内存。将该环境变量设置为内存限制，以 MB 为单位。例如，要限制 2GB 卡上的 Viewport 2.0 仅使用 1GB，则将该环境变量设置为 1024。输出窗口确认已手动设置内存限制。

在同时运行 3D 应用程序的实例中此操作可提高性能，否则将降低性能，因此应谨慎使用。

MAYA_ALLOW_RENDER_LAYER_SWITCHING

将该环境变量设置为 1，这样，在卸载具有简单的着色替代（即，无组件着色、无共享着色网络、无引用的渲染层）的引用对象时，该对象可以将其着色指定正确地维护为当前的渲染层开关。（适用于传统渲染层）。

OCIO

将此环境变量设置为用于颜色管理的 OpenColorIO 配置文件的完整路径和名称。这是一个共享环境变量，由支持 OpenColorIO 的应用程序使用。如果在旧版 SynColor 模式下启动 Maya，则仅支持 OCIO v1 配置文件。

如果设置了 OCIO，则在颜色管理首选项中：
  • 在启动时、新场景中以及场景加载时启用“启用颜色管理”(Enable Color Management)和“使用 OCIO 配置”(Use OCIO configuration)。
  • “OCIO 配置路径”(OCIO config path)设置为由环境变量定义的路径。
  • 各种颜色空间和变换菜单将使用 OpenColorIO 配置文件中定义的选项进行填充。在旧版 SynColor 模式下，内置选项不可用。

如果您使用策略来配置颜色管理，OCIO 环境变量将被覆盖，如使用颜色管理策略（外部首选项文件）中所述。

OCIO_ACTIVE_DISPLAYS

将此环境变量设置为显示的冒号分隔列表，以覆盖 OpenColorIO 配置文件的 active_displays 设置。必须在配置文件中定义这些显示。

OCIO_ACTIVE_VIEWS

将此环境变量设置为冒号分隔的视图列表，以覆盖 OpenColorIO 配置文件的 active_views 设置。必须在配置文件中定义这些视图。

OCIO_INACTIVE_COLORSPACES

将此环境变量设置为冒号分隔的空间列表，以覆盖 OpenColorIO 配置文件的 inactive_colorspaces 设置。

MAYA_VP2_PAUSE_ON_STARTUP

将该环境变量设置为 1 可在启动后暂停 Viewport 2.0。“材质查看器”(Material Viewer)（如果设置为使用硬件渲染器）和其他编辑器（例如，“UV 编辑器”(UV Editor)）也会因此暂停。暂停的编辑器以红色边界显示，且文本“暂停”(Paused)指示其暂停状态。

MAYA_BATCH_DISABLE_UNDO_MANAGER_FLUSH_ON_EXIT

将此环境变量设置为 1，可在对大范围帧烘焙动画的脚本上运行 mayabatch.exe 时提高性能。如果设置了此环境变量，则 mayabatch 会在退出时跳过清除撤消队列。

SHADERFX_CUSTOMUSERPATH

默认情况下，ShaderFX 将自定义元素（例如，组节点、场景和烘焙预览样例）保存至 $HOME 或 \MyDocuments 目录的 maya\ShaderFX 文件夹中。要自定义 ShaderFX 文件夹的位置，请将系统环境变量 SHADERFX_CUSTOMUSERPATH 设置为首选位置。

渲染设置环境变量

MAYA_RENDER_SETUP_GLOBAL_TEMPLATE_PATH

将该环境变量设置为全局渲染设置模板的网络位置。将模板保存到可全局访问的网络位置以供用户共享。

模板是表示渲染设置的 .json 文件；例如，整个渲染设置配置、集合的子集或渲染层及其集合。可以导出模板，然后将其导入到另一个场景中，这样就不必重新创建渲染设置。

Maya 会自动在此目录中搜索可用的模板，并在“渲染设定”(Render Setup)编辑器的快捷菜单中显示适用的选项。请参见导入和导出渲染设置。

MAYA_RENDER_SETUP_GLOBAL_PRESETS_PATH

将该环境变量设置为全局“渲染设置”(Render Settings)预设的网络位置。将预设保存到可全局访问的网络位置以供用户共享。预设可以 .json 文件的形式保存。

Maya 会自动在此目录中搜索可用的预设，并将其显示在“渲染设置”(Render Settings)窗口的“预设 > 加载预设”(Presets > Load Preset)菜单和“渲染设定”(Render Setup)编辑器的 菜单中。请参见将“渲染设置”(Render Settings)作为预设导出和导入。

MAYA_RENDER_SETUP_USE_UNTITLED_COLLECTIONS

此环境变量控制在活动渲染层中创建的新对象是否保存到无标题集合中。将此环境变量设置为 0 可禁用此行为，设置为 1 可启用此行为。此环境变量优先于“渲染设定”(Render Setup)编辑器中的“选项 > 创建新对象后启用‘untitled’集合”(Options > Enable untitled collections when new objects are created)菜单选项。设置此项后，该菜单选项将灰显，系统将应用该环境变量值。请参见由 Maya 自动创建的无标题集合。

MAYA_RENDER_SETUP_INCLUDE_ALL_LIGHTS

此环境变量控制是否所有渲染层默认自动包含灯光。将此环境变量设置为 0 可禁用此行为，设置为 1 可启用此行为。此环境变量优先于“渲染设定”(Render Setup)编辑器中的“选项 > 每个渲染层默认包含所有灯光”(Options > Include all lights in each render layer by default)菜单选项。设置此项后，该菜单选项将灰显，系统将应用该环境变量值。请参见避免默认将灯光添加到层中。

MAYA_RENDER_SETUP_DISPLAY_RS_NODES

此环境变量控制是否在编辑器（如“大纲视图”(Outliner)）中隐藏“渲染设定”(Render Setup)节点（如 renderSetup、renderSetupLayer1、collection1 等）。将此环境变量设置为 0 可禁用此行为，设置为 1 可启用此行为。此环境变量优先于“渲染设定”(Render Setup)编辑器中的“选项 > 在编辑器中显示渲染设置节点”(Options > Display Render Setup nodes in editors)菜单选项。设置此项后，该菜单选项将灰显，系统将应用该环境变量值。请参见在大纲视图和其他编辑器中显示渲染设置节点。

MAYA_RENDER_SETUP_DISABLE_RENDER_SETTINGS_AOVS_EXPORT

此环境变量控制当从“渲染设定”(Render Setup)编辑器选择“文件 > 导出全部”(File > Export All)时场景“渲染设置”(Render Settings)和“AOV”(AOVs)是否随“渲染设定”(Render Setup)节点一起导出。将此环境变量设置为 1 可禁用此行为，设置为 0 可启用此行为。如果要导出模板以使用其渲染设定层、集合和覆盖，但不修改您要向其中导入的场景中的“渲染设置”(Render Settings)或“AOV”(AOVs)，则此功能非常有用。

您还可以通过在“导出所有渲染设定”(Export all render setup)窗口中启用或禁用“导出渲染设置和 AOV”(Export Render Settings and AOVs)选项，在本地设置该行为。如果未设置此环境变量，则默认情况下会启用该行为。

除非明确覆盖，否则环境变量设置优先于 UI 设置。例如，如果将此环境变量设置为 1，然后选择“文件 > 导出全部”(File > Export All)并启用“导出渲染设置和 AOV”(Export Render Settings and AOVs)选项再导出，则场景“渲染设置”(Render Settings)和“AOV”(AOVs)将包含在导出的 .json 文件中。但是，下次选择“文件 > 导出全部”(File > Export All)时，将再次禁用此选项。

请参见导出完整的渲染设定。

MAYA_RENDER_SETUP_ENABLE_WARNING_ICONS

此环境变量控制在缺少集合成员时 图标是否出现在“渲染设定”(Render Setup)编辑器中。将此环境变量设置为 1 表示显示图标，设置为 0 表示隐藏图标。此环境变量优先于“渲染设定”(Render Setup)编辑器中的“选项 > 显示渲染设定警告图标”(Options > Show Render Setup warning icons)菜单选项。设置此项后，该菜单选项将灰显，系统将应用该环境变量值。请参见标识集合中缺少的成员。
