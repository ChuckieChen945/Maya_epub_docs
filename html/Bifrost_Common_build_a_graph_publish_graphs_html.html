以图表形式发布效果

您可以将“Bifrost 浏览器”(Bifrost Browser)中显示的整个效果发布为图表。发布的图表与发布的复合不同：发布的复合是可以包含在图表中的可重用子图表，而发布的图表包含图表的顶层及其与主场景的连接。

发布的图表还可以包含支持文件，例如缩略图和示例场景。

提示：

如果图表具有场景几何体输入（例如网格），并且将图表导入到的场景具有相同名称的对象，则会自动连接该场景。

发布图表：

  1. 在图表顶层，选择“创建 > 发布图表”(Create > Publish Graph)。

  2. 输入所需的信息：

    • 图表名称(Graph name)：图表的名称，其将显示在“Bifrost 浏览器”(Bifrost Browser)中。导入图表时，此项也将用作主场景中 Bifrost 图表节点的默认名称。
    • 图表名称空间(Graph namespace)：所有图表都需要名称空间。
    • 浏览器类别(Browser category)：图表在“Bifrost 浏览器”(Bifrost Browser)中显示时所在的类别。没有类别的图表将不会列出。
    • 作者(Author)：可选的作者姓名。在“Bifrost 浏览器”(Bifrost Browser)中单击信息图标时会显示此项。
    • 描述(Description)：可选描述。单击信息图标时也会显示此项。

  3. 选择可选支持文件。这些文件将被复制到发布位置以便轻松共享。

    • 缩略图(Thumbnail)：要在“Bifrost 浏览器”(Bifrost Browser)中显示的图像。
    • 文档(Documentation)：用于提供有关图表的详细信息的 Markdown 文件。在图表顶层未选择任何节点时显示此项。支持大多数 GitHub 风格 Markdown 格式。
    • 示例场景(Example Scene)：可以从“Bifrost 浏览器”(Bifrost Browser)打开的场景。

  4. 设置发布选项：

    • 包括已连接的材质(Include connected materials)：将图表中引用的任何场景材质保存为单独的场景，导入图表时将自动重新连接这些材质。
    • 保存 sidecar 文件(Save sidecar files)：“与复合相邻”(Adjacent to compound)选项将所有文件存储在一个常用子文件夹中，以便更轻松地压缩为 zip 文件并共享。“在 Autodesk-默认文件夹中”(In Autodesk-default folders)将图表和其他文件放置在单独的文件夹中。

  5. 选择“保存文件”(Save file)以保存文件名和目录。仅存储在下拉框中所列位置之一的复合将在“Bifrost 浏览器”(Bifrost Browser)中可用。除了默认用户位置外，您还可以设置其他复合位置。 默认用户位置为：

    • Windows：\Users\用户名\Autodesk\Bifrost\Compounds
    • macOS：/Users/用户名/Autodesk/Bifrost/Compounds
    • Linux：/home/用户名/Autodesk/Bifrost/Compounds
