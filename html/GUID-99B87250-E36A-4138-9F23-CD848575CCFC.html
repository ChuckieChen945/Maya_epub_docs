节点和属性名称中的扩展 ASCII 字符以及场景可移植性

在使用 CP-1252 的 Windows 和使用 Latin iso8859-1 的 Linux 上，Maya 支持使用扩展 ASCII 范围内单字节字符的节点和属性名称。这些扩展 ASCII 字符集包括带重音符的字母，以及其他变音符号和双音符号（例如 é 和 ß）。

日语 Hankaku 字符不受支持。

在 Windows 上，系统区域设置必须设置为 CP-1252 语言，例如英语、法语或德语。

在 Linux 上，LANG 环境变量必须设置为与 Latin iso8859-1 兼容的区域设置，例如美国英语 (en_US.iso88591)、加拿大法语 (fr_CA.iso88591) 或奥地利德语 (de_AT.iso88591)：
export LANG=en_US.iso88591

必须在启动 Maya 之前设置 LANG 变量。

在 macOS 上，Maya 不支持在节点和属性名称中使用扩展 ASCII 字符集。在采用多字节区域设置（例如 UTF-8）运行的 Linux 上，它也不支持扩展 ASCII 字符集。

如果在不受支持的环境中打开包含扩展 ASCII 字符集中的字符的场景文件，可能会导致数据丢失。

若场景文件的节点或属性名称中包含较低 ASCII 范围之外的字符，可能会在平台之间移动时出现问题。
