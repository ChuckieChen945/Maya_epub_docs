2.0.2.1

Release Date

September 18, 2017

This version uses the Arnold 5.0.1.3 core.

DOWNLOADS

  • https://manage.autodesk.com/products/

This version includes the following fixes:

  • MtoA environment variables could conflict with other renderers, and prevent them from loading properly
  • Fixed XGen rendered with card primitives
  • When Cameras didn't use global shutter settings, motion blur was incorrect
  • ARV settings saved in user preferences could fail and be reset
  • Fixed crash in ARV when snapshots folder was saved and reloaded
  • When ARV was opened without MtoA loaded, it wasn't possible to load MtoA anymore without restarting Maya
  • Fixed IPR update issue with per-face shading changes
  • Fixed IPR update issue on stand-ins material assignment
  • Fixed IPR update issue on custom procedurals
  • Fixed bug in Render to Texture used with procedurals
  • Exposed UV sets parameter in Render to Texture
  • aiColorJitter couldn't enter negatives values into gain or saturation min/max values
  • Restored Arnold parameters in the Attribute Spreadsheet
  • Prevent Maya errors when an old scene saved with Arnold 4 was opened
  • Remove warning caused by AA_sample_clamp_affect_aovs translation
  • XGen was exported with random names and "id" values
