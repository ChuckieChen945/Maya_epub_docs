interpret_auto_port_as_scalar

将各种输入转化为 array<float> 类型。使用此工具节点可以构建支持多种输入类型的可重用复合，以修改几何对象上的点或其他组件数据。例如，可以使用它以各种方式指定逐点权重。

要使用此节点，请将 interpreted_scalar_port 连接到复合内类型为 auto 的输入，然后在每个组件计算中使用输出 data。

支持以下多种可能的输入类型：

  • 单个 float 解释为元素的常量值。
  • array<float> 解释为每个元素（例如，每个点）的值。数组将会传递，但可调整大小以匹配 geometry 输入上的元素数。
  • array<bool> 解释为每点的 0 (false) 或 1 (true) 值。
  • string 解释为现有几何特性的名称，并使用其值。
  • ScalarField 将在几何体元素的每个位置进行采样。
  • array<long> 指定要为其指定值 1 的索引。对于其他索引，则指定值 0。
  • array<bool> 与 array<float> 类似，其中数据是每个元素特定的，但值 1 表示 true，值 0 表示 false。

输入

geometry

要在其中解释输入的几何上下文。例如，使用默认设置和基于点的几何体时，将在点位置对场进行采样。输出与几何体中的元素（如点）数匹配。

interpreted_scalar_port

将此端口连接到复合的 auto 输入，以支持不同类型的输入。从这个端口创建新端口时，还会向该端口添加解释的自动类型调整。

default_value

在以下情况下要使用的默认值：

  • 未将任何内容连接到 interpreted_scalar_port。
  • 连接了 array<float>，但数组太小。
  • 连接了空标量场。
  • 当 string 指定特性但该特性不可用时。

indices

如果不需要输入几何体中每个元素的值，应连接需要其数据的组件索引的数组。如果索引无效，则将返回默认值。如果提供索引，则输出 data 数组的大小与此数组匹配，而不是与组件中的元素数匹配。

geometry_component

如果数据上下文不是默认的 point_component，则可以在此处选择其他组件。这是可确定输出 data 数组大小的组件。

position_property

如果对象不使用默认的 point_position 特性，则更改此字符串。这仅用于对场进行采样（如果已连接）。

输出

data

将输入 interpreted_scalar_port 转化为 array<float> 后所得的结果。此数组与 geometry 中的元素数匹配，或与 indices 输入（如果已连接）的大小匹配。

has_data

指示是否已将有效输入连接到 interpreted_scalar_port。如果输入为 float 值、由 float 值组成的非空数组、由 long 值组成的数组、非空标量场或作为 geometry 上有效特性的名称的字符串，则此项返回 true。
