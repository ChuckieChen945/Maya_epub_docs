update_transform_tree

使用各种更新方法更新解释为展平的变换树的现有 array<Transform>。

输入

parent_indices

包含父索引值的 array<long>，定义变换父对象在输入数组中的位置。如果索引为负，则从辅助 parent_transforms 数组中选择父对象。

变换的排序相对灵活，但必须确保变换的父对象在数据数组中的位置位于变换之前。

如果变换是根或是隔离的，则其父索引值必须等于 invalid_index 节点返回的值。

transforms

要更新的 array<Transform>。

parent_transforms

变换树的辅助父对象变换（如果有）。

可以在主变换计算中使用辅助父对象，方法是在 parent_indices 中使用负索引值。值为 -1 时将使用第一个辅助父对象，值为 -2 时将使用第二个辅助父对象，依此类推。

update_matrices

用于更新 transforms 树的变换矩阵数组。这些矩阵应反映选定的 update_method，以便正确解释。

transforms_to_update

包含要变换的关节的索引值的 array<long>。

如果为空，则将更新所有变换，假定 update_matrices 数组大小与输入 transforms 的大小匹配。

update_method

定义应如何解释 update_matrices 输入以及如何更新变换。

  • From Pivot：使用新的枢轴矩阵重新计算所有变换矩阵分量。

  • From Operator：使用新的运算符矩阵重新计算非枢轴变换矩阵分量。

  • From Local：计算运算符矩阵，并直接使用局部空间动画结果重新计算移动矩阵。

  • From World：计算运算符矩阵，并直接使用世界空间动画结果重新计算移动矩阵。在使用反向运动学时，通常使用此方法。

overwrite

如果启用此项，则使用输入 update_matrices 覆盖选定的变换分量。否则，将 update_matrices 乘以从 update_method 选择的现有变换分量。

输出

out_transforms

更新的 transforms。
