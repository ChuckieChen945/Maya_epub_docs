force_pull_port

此复合将强制图表对数据流进行提取和解算。它包含一个通过元素和要强制图表进行解算的流。

此节点具有多种用途：

  • 反馈端口：从反馈端口提取数据可确保对这些端口及其上游的所有对象进行解算。这特别适用于同一节点上存在多个反馈端口的情况。
  • 模拟：仅在提取模拟的数据时，模拟才会前进。对于链状模拟，第一个模拟可能从第 1 帧开始，第二个模拟可能从第 10 帧开始。仍需要从第 1 帧开始对第一个模拟进行解算，这可用于确保已对其进行提取。
  • 文件 IO 和其他额外作用：force_pull_port 可用于确保对 FileIO 节点进行解算并写入其文件。

输入

value

这是通过端口。此处插入的任何对象都通过复合，没有任何变化。如果在图表的常规操作中未进行提取，则此复合将不起作用。

pull

这是要解算的流。如果正在提取节点（请参见 value 端口），则将强制图表解算插入此处的任何对象。

输出

out_value

复合的输出，即通过端口 value 的另一端。数据保持不变。
