颜色管理入门

颜色管理为您提供了场景线性工作流的优势，同时仍可在视口和“渲染视图”(Render View)中显示适用于您的显示的预览。默认颜色管理系统为 OpenColorIO v2 (OCIO v2)。

注：

有关 OCIO 的背景信息，请访问 opencolorio.org。

设置颜色管理

  1. 打开颜色管理首选项并确保颜色管理处于启用状态。

    通常，默认情况下对新场景启用颜色管理，但是可以在新建场景选项中更改此行为。

  2. 如果项目使用特定的 OCIO 配置，请将“OCIO 配置路径”(OCIO Config Path)设置为磁盘上的完整路径和文件名。
    注： 如果在启动 Maya 时遇到与颜色管理相关的延迟，请确保 OCIO 配置文件包含 environment 部分。该部分是否为空无关紧要，例如：
    environment:
  {}

    如果在启动 Maya 之前已将 OCIO 环境变量设置为配置文件的名称和路径，则“OCIO 配置路径”(OCIO Config Path)为只读。

  3. 将“渲染空间”(Rendering Space)设置为用于渲染的工作颜色空间。请参见选择渲染空间。
  4. 确保“显示”(Display)适合您的显示设备。
  5. 将“视图”(View)设置为最适合您的项目的预览。请参见选择用于预览的视图。
  6. 在“输入颜色空间规则”(Input Color Space Rules)中，（可选）设置用于为纹理指定颜色空间的规则。至少应设置“默认”(Default)规则，以指定大多数图像输入使用的颜色空间。可以基于文件命名约定添加其他规则。请参见定义规则以为输入文件指定颜色空间。

    默认情况下，会将由规则确定的颜色空间用作新的 2D 文件纹理和其他图像输入的初始设置。您仍可以按为纹理和其他图像输入指定颜色空间中所述单独更改每个设置。

  7. 如果要将颜色管理应用于已经有材质、纹理和灯光的现有场景，请参见将颜色管理应用于现有场景。
  8. 有关您可能需要进行的某些查看和渲染设置，请参见重要的颜色管理设置。
注： 在使用 Turtle 插件烘焙纹理颜色时，不应当启用颜色管理。如果执行此操作，则视口预览将不准确。

添加已管理颜色的图像输入

将 2D 文件纹理和其他图像输入添加到场景中时，需要指定其输入“颜色空间”(Color Space)（如果与规则指定的不同）。有关详细信息，请参见为纹理和其他图像输入指定颜色空间。

预览场景

当您工作时，视图变换和显示变换将在视口和“渲染视图”(Render View)中提供最终颜色预览。但是，您可以暂时关闭视图变换和显示变换，或者选择其他视图用于诊断或其他用途。请参见预览已管理颜色的场景。

对于播放预览，“将输出变换应用于播放预览”(Apply Output Transform to Playblast)选项应用选定变换以便从渲染空间转化。通常应该启用此选项，以允许使用未对场景线性图像应用相应视图变换的设备或查看软件来播放结果。

渲染场景

渲染场景时，可以选择在输出变换中烘焙，具体取决于要渲染一个预览还是渲染图像以供合成等进一步处理。请参见渲染已管理颜色的场景。

旧版颜色管理功能

出于兼容考虑，defaultViewColorManager 节点在场景中仍然可用。但是，不使用其选项。
