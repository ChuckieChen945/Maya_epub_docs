Arnold for Maya 5.4.5 插件

了解 MtoA 5.4.5 for Maya 2025.3 中的新功能。

此功能版本提供对 OpenPBR 曲面的支持、改进的体积渲染、对 Apple Metal 和 AMD GPU 的 OIDN 支持，以及许多其他改进和错误修复。

有关最近添加的功能和错误修复的完整列表，请参见 Arnold for Maya 发行说明。

OpenPBR 曲面

新的 OpenPBR 曲面着色器实施 OpenPBR 规范。OpenPBR 曲面是标准曲面的演变形式，是与 Adobe 和 Academy Software Foundation 协作开发的。它引入了许多改进，尤其是更好的光泽/柔化模型以及改进的金属反射率参数化。(ARNOLD-13385)

OpenPBR 工作流工具

MtoA 包含多种工具，有助于将 aiStandardSurface 着色器转化为 OpenPBR 曲面 (MTOA-1829)

体积中的散射扩散

standard_volume 中的两个新参数改进了灯光在体积中的散射方式。使用这些参数有助于在不增加能量的情况下，以较少的体积反弹获得高散射外观。(ARNOLD-14482)

  • scatter_diffusion 可加快体积中的散射。
  • scatter_diffusion_roughness 可进一步加快各向异性体积上的散射，并有助于以较少的反弹获得逼真的云。
  • 增益曲线和偏移曲线可以进一步调整两个参数的效果

随机体积插值(Stochastic Volume Interpolation)

AiVolumeSampleXXX() 调用的三元三次插值模式（如 standard_volume 着色器使用的模式）现在使用随机插值的方法，这可极大地减少在每个采样调用中读取的体素数据量。在测试中，我们看到某些场景的速度提高了多达 1.7 倍。请注意，此方法并不总能生成完全匹配的图像，并且可以通过 stochastic_volume_interpolation 渲染选项（默认设置为打开）来进行切换。(ARNOLD-3856)

改进了 Intel Open Image Denoise 2.3.0 的质量

更新后的 Intel 降噪器可提高降噪质量，并在具有镜面反射和透射的场景中生成更清晰的结果，同时减少瑕疵。有关此版本中更改的详细信息，请参见 OIDN 发行说明 (ARNOLD-15059)。

替代对象视口显示颜色

可以在 aiStandIn 节点上设置自定义颜色，以通过属性“对象显示/绘制覆盖/启用覆盖”(Object Display/Draw Override/Enable Overrides)控制它们在视口中的显示方式

球体和长方体体积已添加到 aiVolume

aiVolume 中添加了一个新属性，用于将体积形状设置为长方体或球体，默认为现有的文件模式 (MTOA-247)

支持摄影机“backgroundColor”

现在支持使用 Maya 摄影机属性“backgroundColor”来更改当前渲染的背景颜色 (MTOA-1727)

Apple Metal 和 AMD GPU 的 OIDN 支持

Intel OIDN 降噪器现在在 macOS 上支持所有 Apple Metal GPU（M1 及更高版本），在 Windows 上支持使用 RDNA2（仅限 Navi 21）和 RDNA3 (Navi 3x) 架构的 AMD GPU。这样可以显著加快降噪速度：在 M1 Max 笔记本电脑上，降噪速度提高了 5-8 倍。（ARNOLD-14855、ARNOLD-14856）

GPU 对灯光链接的多灯光支持

Arnold GPU 现在对链接到形状的灯光组使用“全局灯光采样”(Global Light Sampling)。这可以提高具有灯光链接的场景的渲染性能。例如，现在，当使用“全局灯光采样”(Global Light Sampling)时，ALab 场景渲染速度是原来的两倍。(ARNOLD-15050)

GPU 缩短了在具有许多节点的场景中呈现第一个像素的时间

借助 Arnold GPU，在具有许多节点的场景，缩短呈现第一个像素的时间。例如，在具有 1,000,000 个实例的简单场景中，呈现第一个像素的时间缩短了 18%。(ARNOLD-13652)

GPU 上的运动向量 AOV

在 GPU 上渲染时，现在支持运动向量 AOV。(ARNOLD-11046)

改进了渐变着色器

3D 渐变模式

渐变着色器有三种新模式：3d_linear、3d_spherical 和 3d_cylindrical。这些模式根据着色点与用户定义的参照点或线之间的距离来计算渐变输入。（ARNOLD-15051、ARNOLD-15196）

3d_linear 3d_cylindrical 3d_spherical

渐变偏移

渐变着色器现在可以对输入值进行偏移，从而允许您向渐变添加噪波。（ARNOLD-15052、ARNOLD-15197）

worley 噪波 (3d_spherical) cell_noise (u) alligator 噪波 (v)

热量贴图成像器模式

色调映射成像器现在具有热量贴图模式，可基于查找的 AOV 数量创建光谱。颜色范围介于蓝色（最小值）和红色（最大值）之间。(ARNOLD-15123)

程序中体积的视口表示

现在，体积可以显示在视口中。当前，仅当在 MtoA 中加载程序 (aiStandIn) 时支持此功能。(ARNOLD-14382)

Arnold 渲染视图增强功能

  • 使用 MEL 访问“保存图像”(Save Image)选项：Arnold 渲染视图中的“保存图像”(Save Image)选项现在可通过 MEL arnoldRenderView 命令访问 (ARNOLD-15083)
  • 通过单击选择范围外的位置来取消选择快照：单击 Arnold 渲染视图快照面板中的空白区域，将取消选择选定的快照 (ARNOLD-15080)
  • 快照文件夹的用户首选项：快照文件夹有一个新的用户首选项。您可以通过设置环境变量 ARNOLD_SNAPSHOTS_FOLDER 来覆盖此选项 (MTOA-1934)

其他增强功能

  • 改进了非渐进渲染中的 AOV 内存使用量：使用长方体过滤器的非渐进自适应渲染现在使用的内存更少。使用以下过滤器时，非自适应渲染中的内存使用量也会减少：gaussian_filter、triangle_filter、sinc_filter、catrom_filter、mitnet_filter、blackman_harris_filter。例如，在 1080p 渲染中，使用 AA_samples_max 50 和 6 AOV 的自适应，AOV 使用的内存量从 14 GB 减少到 730 MB (-96%)。另请注意，与先前版本不同，随着 AA_samples_max 的增加，AOV 内存使用量现在会保持恒定，而在先前版本中，内存使用量会呈二次方增加。(ARNOLD-10160)
  • 提高了 Alembic 程序的效率：Alembic 程序现在可以并行读取数据，从而优化对包含许多对象的大型 Alembic 文件的初始化时间。我们发现，在大型制作场景中，启动时间从 15 分钟变为 4 秒。(ARNOLD-14796)
  • 间接漫反射和间接镜面反射参数现在可在 standard_surface 和 openpbr_surface 着色器中链接：standard_surface 和 openpbr_surface 着色器的 indirect_diffuse 和 indirect_specular 现在支持着色器链接。这可以在不需要间接照明的情况下进行更好的优化。(ARNOLD-14204)
  • 三平面 Nref 用户参数：三平面着色器现在接受 Nref 用户参数，以允许在 Pref 模式下指定引用法线。(ARNOLD-15063)
  • 用于 OpenImageIO 子图像的 AOV 名称：多部分文件以前为层/子图像使用默认的子图像命名。Arnold 现在将 AOV 输出的名称用于相应的子图像。(ARNOLD-15053)
  • 三角形细分内存使用情况统计信息分组在多边形网格内存使用情况下：Arnold 日志现在在“已用 CPU 内存峰值”和“已用 GPU 内存峰值”中同时列出三角形细分和多边形网格内存使用情况。(ARNOLD-15194)
  • 将自发光亮度从 nits 缩放到 Arnold 单位：已将 nits_per_unit 添加到选项节点，OpenPBR 曲面的 emission_luminance（单位为 nits）将按此 nits_per_unit 值（默认为 1000）在内部缩小，以将其转换为 Arnold 单位。这可确保 emission_luminance 的默认软最大值 1000 nits 与自发光 1 时“标准曲面”(Standard Surface)的先前行为相对应。(ARNOLD-15364)
  • GPU 上的启动时间更短：我们进行了多次优化，以缩短 GPU 上的启动和场景更新时间。这些改进在包含许多对象的场景中更为明显。Windows 上的启动时间现在与 Linux 上的启动时间不相上下。与先前版本相比，在具有 100 万个实例的场景中，进行了以下改进。（ARNOLD-15271、ARNOLD-15279、ARNOLD-15310）
    7.3.3.1 7.3.4.0 改进
    GPU 更新时间 (Windows) 204s 17s 12x
    GPU 更新时间 (Linux) 37s 18s 2x
  • 在 OSL 着色器中更快地查找纹理不透明度：在不透明度输入中使用的 OSL 着色器（例如，OpenPBR 曲面的几何体不透明度）现在可以更快地读取不透明度遮罩纹理，并且占用的内存更小。(ARNOLD-15340)
  • 使用 kick 改进了节点和参数自检
    • kick -info <node> 现在按 UI 组元数据或参数是输入还是输出对节点参数进行分组。(ARNOLD-14198)

  • OSL 着色器更改的控制柄更新：使用交互式渲染模式时会存储有关链接到 OSL 着色器的着色器的信息，因此这些着色器中的任何更改都将导致 OSL 着色器自动更新。(ARNOLD-14983)
