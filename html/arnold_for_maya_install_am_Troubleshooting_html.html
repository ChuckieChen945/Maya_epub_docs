疑难解答

如果您在按照安装说明进行操作后仍然遇到问题，请尝试以下步骤：

如果安装程序提示 MtoA 已安装但无法卸载它

当检测到已安装了先前版本时，安装程序会尝试运行卸载程序卸载之前的安装。但是，如果出于某些原因，您在运行安装程序之前移动、删除或重命名了旧版安装文件夹，则无法找到旧版卸载程序。这时，您需要自己找到并运行旧版安装的 Uninstall.exe 程序（如果仍有的话），而无法通过在 Windows“开始”菜单的 MtoA 文件夹中找到卸载选项或使用 Windows 中的“添加/删除程序”执行卸载。

如果您在 Maya 插件管理器中看不到 MtoA 插件

  1. 检查 mtoa.mod 文件是否位于以下文件夹中（假设主机是 Maya 2023）：

    C:\Users\Documents\maya\2023-x64\modules

    并包含以下文本： 

     + mtoa any C:\solidangle\mtoadeploy\2023

  1. 尝试在系统中设置以下环境变量（从理论上说，Maya 应在加载模块时自动添加这些变量）：

     MAYA_PLUG_IN_PATH=C:\solidangle\mtoadeploy\2023\plug-ins 
     MAYA_SCRIPT_PATH=C:\solidangle\mtoadeploy\2023\scripts 
     XBMLANGPATH=C:\solidangle\mtoadeploy\2023\icons

如果您可以看到 MtoA 插件，但它无法加载

在这种情况下，或者如果在尝试加载时该插件发生崩溃，请尝试以下步骤：

设置以下环境变量：

 PYTHONPATH=C:\solidangle\mtoadeploy\2023\scripts

移除（并备份）Maya 首选项文件夹：

 C:\Users\<User_Name>\Documents\maya\2023-x64\prefs

使用全新 Maya 安装尝试加载 MtoA。

使用全新或最近配置的计算机尝试加载 MtoA。

如果您可以看到 MtoA 插件，但它适用于其他版本的 Maya

检查系统“环境变量”或 Maya.env 路径中是否并无条目指向其他 Maya 版本的 MtoA 路径。以前的 Maya 配置可能是第一次出现在该文件中。考虑到 Maya.env 仅用于 Maya 2012 及更高版本对应的 MtoA，因此其内部不应存在任何 MtoA 路径跟踪。
