tag_scope

用于可视化和分析几何体的标记元素的诊断复合。

输出表示几何体的对象数组，可以输出这些对象以可视化标记。

输入

geometry

要分析的几何体。

tag_data

解释为标记数据的自动端口。此端口接受与 interpret_auto_port_as_component_tag 相同的输入：

  • array<bool> 解释为每个组件的值。数组将会传递，但可调整大小以匹配 geometry 输入上的元素数。
  • array<long> 或 array<uint> 解释为组件索引。
  • string 解释为现有标记的名称或标记表达式。有关标记表达式语法的更多详细信息，请参见有关 resolve_tag_expression 的文档。

weights

每个元素的权重数组。数组大小应等于指定的 component_type 的元素数。假定权重归一化到 [0,1] 范围内。

geometry_component

标记的组件类型。

颜色部分

tag_color、weight_color

输入 weights 用于在 tag_color 与 weight_color 之间进行插值

形状部分

point_size

用于点显示的圆的世界空间大小。

strand_width

用于股显示的带状的世界空间大小。

offset_amount

亮显的面相对于曲面偏移的量，用于面和面顶点显示。

triangle_size

用于面顶点显示的三角形的大小。介于 [0, 0.5] 之间的归一化值。值为 0.5 表示三角形边位于标记的面顶点与面中相邻面顶点的中间位置。

输出

diagnostic_geometry

表示用户启用的可视化数据的诊断几何体数组。此数组将包含点、股和网格几何体的混合。
