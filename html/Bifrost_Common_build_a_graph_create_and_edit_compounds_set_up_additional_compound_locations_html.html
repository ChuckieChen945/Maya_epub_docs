设置其他复合和图表位置

默认情况下，您创建的复合和图表将发布到您的 Bifrost 用户位置。但是，您可以根据需要设置其他目录位置。执行此操作后，您可以在发布时选择这些位置。这些额外位置中的任何复合和图表会像任何其他复合和图表一样显示在“选项卡”(Tab)菜单或 Bifrost 浏览器中。

  1. 在您要将复合发布到的文件夹的上一级目录中创建 JSON 配置文件。

  2. 将复合文件夹的相对路径添加到配置文件的 jsonLibs 部分。

  3. 将配置文件的完整路径（包括文件名和扩展名）添加到 BIFROST_LIB_CONFIG_FILES 环境变量。

  4. 重新启动 Bifrost 插件。

请注意：

  • 路径不能包含下列任何字符：
    : * ? " < > |
  • 如果在指定路径下找到的符号链接指向配置文件本身所在位置下的层级之外，则不会遍历这些链接。例如，如果 path/myConfig.json 包含相对路径 folder1，而 path/folder1 包含指向 path/folder2 的符号链接，则将跟随该符号链接，但不会跟随指向 other_path/folder2 的符号链接。
    • 空路径 ("") 将被忽略。这在开发复合时非常有用，因为您可以通过修改路径而不是删除整个 {"path": ... "files": ... }, 部分来移除库。

示例配置文件

如果只是为复合和图表定义其他文件夹，则配置文件可以像此示例一样简单。请注意，路径是相对于配置文件本身位置的路径。

本示例在与配置文件相同的目录中定义两个子文件夹，compounds 和 experiments。请注意，libraryName 和 libraryVersion 条目用于在加载复合文件时记录信息和错误。

{
  "AminoConfigurations": [
    {
      "libraryName": "My_Compounds",
      "libraryVersion": "3.1.4",
      "jsonLibs": [
        {
          "path": "compounds",
          "files": [ ]
        },
        {
          "path": "experiments",
          "files": [ ]
        }
      ]
    }
  ]
}
