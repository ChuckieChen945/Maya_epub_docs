5.4.1.1

24 April 2024

MtoA 5.4.1.1 uses Arnold 7.3.1.1. See also the MtoA 5.4.1 release notes.

Installation

  • Download Arnold for Maya from your Autodesk Account. See Download Arnold for more information about downloading MtoA from your Autodesk Account.

  • Follow these installation instructions.

Bug Fixes

  • MTOA-1240 - Import Shaders does not create shadingEngine for volume shader
  • MTOA-1842 - Maya Arnold Viewport is not accounting for some camera film back settings causing geometry to be offset from the render
  • MTOA-1843 - Error message with XGen when creating clumps from guides
  • MTOA-1851 - Instancer uses scale of instanced object wrongly
  • MTOA-1877 - If a floating point value is used for Frame Rate in Maya Arnold USD will export as framesPerSecond 0
  • MTOA-1879 - TX Manager crash on second failed conversion
  • MTOA-1881 - Non-render errors and warnings printed to script editor
  • MTOA-1891 - Error when assigning material on AiVolume node
  • MTOA-1896 - aiLayerRgba defaults to false for layer enable1 option
  • MTOA-1901 - Batch render fails on macOS
  • MTOA-1902 - Batch renders fail to open a port
  • MTOA-1903 - Windows cmd prompt launched on batch render
  • MTOA-1905 - ArnoldCreateDenoiser optionVar reset to true on launch
  • ARNOLD-6189 - strings with trailing backslashes result in corrupt .ass files: .ass writer now converts those to forward slashes
  • ARNOLD-11222 - Rare hang when reading .ass files that have certain sizes
  • ARNOLD-14899 - Random crash while deleting nodes from procedural_update
  • ARNOLD-14949 - Update expired lmutil certificate
  • ARNOLD-14698 - [MaterialX] Crash when reading document containing unknown node definitions
  • ARNOLD-14883 - [MaterialX] Crash in LookdevX/MaterialX when visualizing the dot node in the viewport

System Requirements

  • Maya 2023, 2024, or 2025
  • Windows 10 or later, with the Visual Studio 2019 redistributable.
  • Linux with at least glibc 2.17 and libstdc++ 4.8.5 (gcc 4.8.5). This is equivalent to RHEL/CentOS 7.
  • x86-64 CPUs need to support the SSE4.1 instruction set.
  • macOS 10.13 or later, macOS 11 and later for Maya 2024
  • Apple Mac models with M series chips:
    • Natively supported by Arnold for Maya 2024
    • Supported under Rosetta 2 mode for older versions of Maya
  • GPU rendering works on Windows and Linux only and requires an NVIDIA GPU with the Maxwell architecture or later.
    • On Linux, we recommend 535.104 or higher drivers.
    • On Windows, we recommend 537.13 or higher drivers.
  • Optix™ denoiser requires an NVidia GPU with CUDA™ Compute Capability 5.0 and above.
