将引用编辑指定给脱机文件

您可以通过将引用编辑文件（.editMA 或 .editMB）中包含的编辑引用或导入到场景中，以将编辑指定给被引用的对象。引用编辑文件使用“导出到脱机文件”(Export to Offline File)命令创建。请参见将引用编辑导出到脱机文件。

通过在编辑文件中匹配每个节点名称、属性，Maya 将编辑指定给其所适用的文件。例如，可以导出场景 sphere.ma 中的 pSphere1.translateX 的引用编辑。该编辑将在引用文件中保存为 <main>: pSphere1.translateX。然后可以将该编辑指定给场景 ball.ma，以便编辑球：pSphere1.translateX。

注：

悬挂连接（到非导出节点的连接）和引用编辑将保存到具有占位符名称空间的编辑文件。应用这些编辑时，Maya 将使用应用当前文件的引用的名称空间或导入到主场景中的“:”来替换占位符。

将编辑指定给被引用文件

将脱机编辑应用于被引用对象

  1. 通过执行下列操作之一来访问“指定脱机文件选项”(Assign Offline File Options)窗口：
    • 在“大纲视图”(Outliner)中，在要为其应用引用编辑的被引用文件上单击鼠标右键，然后从显示的菜单中选择“指定脱机文件”(Assign Offline File) > 。
    • 在“引用编辑器”(Reference Editor)（“文件 > 引用编辑器”(File > Reference Editor)）中，选择要为其应用编辑的被引用文件，然后选择“文件 > 指定脱机文件”(File > Assign Offline File) > 。
  2. 在“指定脱机文件选项”(Assign Offline File Options)中，选择“编辑 > 重置设置”(Edit > Reset Settings)。
  3. 若要将编辑保持在各自的文件中，请为“引用”(Reference)设定“应用方式”(Apply via)。或者，若要将编辑保存到主场景中，请将其设定到“导入”(Import)。
  4. 单击“指定脱机文件”(Assign Offline File)。
  5. 导航到包含编辑文件的文件夹并选择它。

    此时将显示有效的编辑文件，扩展名为 .editMA 或 .editMB。

  6. 单击“指定脱机文件”(Assign Offline File)。

    编辑将应用于被引用文件。

如果选择通过引用来应用编辑，则脱机编辑文件将显示在引用列表底部的“引用编辑器”(Reference Editor)中。编辑文件的引用节点也显示在“大纲视图”(Outliner)中。

如果在“大纲视图”(Outliner)中看不到引用节点，请确保“大纲视图”(Outliner) 显示菜单中的“引用节点”(Reference Nodes)处于启用状态。

与其他被引用节点一样，您可以加载和卸载引用编辑文件，以查看具有和不具有引用编辑过的场景。

如果选择通过导入来应用编辑、脱机编辑将烘焙到场景中。“引用编辑器”(Reference Editor)中没有为它们显示条目，但您可以在“引用编辑”(Reference Edits)窗口中移除它们。请参见移除引用编辑。

如果对主场景中编辑过的属性进行进一步的更改，然后卸载并通过“大纲视图”(Outliner)引用菜单或“引用编辑器”(Reference Editor)重新加载引用编辑，Maya 将恢复这些编辑过的属性的原始值。

将编辑指定给主场景

也可以将引用编辑文件指定给主场景中的对象。

将脱机编辑指定给主场景中的对象

  1. 通过执行下列操作之一来访问“指定脱机文件选项”(Assign Offline File Options)窗口：
    • 在“大纲视图”(Outliner)中，在要为其应用引用编辑的被引用文件上单击鼠标右键，然后从显示的菜单中选择“指定脱机文件”(Assign Offline File) > 。
    • 在“引用编辑器”(Reference Editor)（“文件 > 引用编辑器”(File > Reference Editor)）中，选择要为其应用编辑的被引用文件，然后选择“文件 > 指定脱机文件”(File > Assign Offline File) > 。

    此时将显示“指定脱机文件选项”(Assign Offline File Options)窗口。

  2. 在“常规选项”(General Options)区域中，将“应用方式”(Apply via)设置为“引用”(Reference)。
  3. 若要将编辑应用于主场景，同时还保持引用这些编辑，请启用“为主场景指定脱机编辑”(Assign offline edits to main scene)。否则，请禁用该选项，以便仅保持对编辑的引用。
  4. 单击“指定脱机文件”(Assign Offline File)。

    此时将显示文件浏览器。

  5. 导航到包含编辑文件的文件夹并选择它。

    此时将显示有效的编辑文件，扩展名为 .editMA 或 .editMB。

  6. 单击“指定脱机文件”(Assign Offline File)。

    编辑将应用于主场景。如果检查引用编辑器，可以看到被引用编辑文件。

在任一情况下，均可以输入编辑，以便不对这些编辑创建引用条目。

通过导入将编辑指定给主场景

  1. 通过执行下列操作之一来访问“指定脱机文件选项”(Assign Offline File Options)窗口：
    • 在“大纲视图”(Outliner)中，在要为其应用引用编辑的被引用文件上单击鼠标右键，然后从显示的菜单中选择“指定脱机文件”(Assign Offline File) > 。
    • 在“引用编辑器”(Reference Editor)（“文件 > 引用编辑器”(File > Reference Editor)）中，选择要为其应用编辑的被引用文件，然后选择“文件 > 指定脱机文件”(File > Assign Offline File) > 。

    此时将显示“指定脱机文件选项”(Assign Offline File Options)窗口。

  2. 选择“编辑 > 重置设置”(Edit > Reset Settings)。
  3. 在“常规选项”(General Options)区域中，将“应用方式”(Apply via)设置为“导入”(Import)。
  4. 单击“指定脱机文件”(Assign Offline File)。

    此时将显示文件浏览器。

  5. 导航到包含编辑文件的文件夹并选择它。
  6. 单击“指定脱机文件”(Assign Offline File)。

    编辑将应用于主场景。与通过引用应用编辑不同，“引用编辑器”(Reference Editor)中不显示编辑的条目，因此不能禁用或撤销这些编辑。

重映射名称空间

根据项目设置的方式和文件的位置，Maya 可能会要求您重映射名称空间，以正确地指定编辑。

重映射名称空间

  1. 指定带有不匹配名称空间的脱机文件。

    此时将出现“重映射缺少的名称空间”(Remap Missing Namespace)窗口。

    该窗口的左侧显示将要发生的自动重映射。右侧显示无相应映射的名称空间。

    通常，<root> 指根名称空间，而 <main> 指当前工作区。

  2. 在右侧选择名称空间，然后单击“贴图”(Map)，以便将其映射到场景中的特定名称空间。否则，请单击“跳过”(Skip)，以便将名称空间放入当前名称空间。
